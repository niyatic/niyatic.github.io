Honest caveats

This cell did not meet the n=95 benchmark bar. It is published as an honest partial run so the failure mode is on the record:
  • Quota wall. The free-tier key allows ~20 requests/model/day for Gemini 2.5 and 0 for gemini-2.5-pro. 95×3 = 285 calls cannot be made. 13 genuine extractions were obtained before exhaustion; the rest are honest {"_error":...} or missing files. Nothing was fabricated or zero-filled.
  • gemini-2.5-pro produced zero outputs — limit: 0 on this key. It is shown as NOT RUN, never estimated.
  • Two real outputs were lost to JSON truncation (docs 1, 2 on Flash-Lite): very large invoices (122 / 40 line items) exceeded the pinned 16,384-token output cap mid-JSON. Recorded honestly as parse-failure errors, not retried (no quota).
  • Decoding was pinned identically across models: temperature=0, max_output_tokens=16384, identical fixed prompt + schema as claude-compare/EXTRACTION-SPEC.md.
  • n=4 / n=6 / n=7 are not statistically powered. No overall-F1 CI is quoted because at this n it would span nearly [0,1]. Treat all rankings here as indicative only.
  • Cross-family comparison with HyperAPI / Claude / OpenAI cells is indicative — different harnesses, different n's, not a controlled bake-off.
