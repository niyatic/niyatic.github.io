Dataset summary

SAVIOR-Bench v1 — 509 expert-annotated financial documents for OCR transcription, layout fidelity (PaIRS), and parse-intent extraction. Inter-annotator agreement 0.761 (fuzzy string similarity, 187 double- annotated pairs).

Document classes: invoices, packing slips, forms, multilingual scans, low-quality PDFs, complex tables.

Tasks & metrics

  • OCR (n=509): word-level multiset recall / precision; PaIRS

layout fidelity (prompt-controlled models only; NPC = format-compat proxy).

  • Parse-intent: F1 ↑ / WER ↓ (n not reported in attested source).
  • Decoding (frozen): temp 0.1, top-p 0.9, max_tokens 2048, seed 20260516.

Model card — SAVIOR-Train fine-tunes (Hyperbots-IP)

Qwen2.5-VL-7B-F, Qwen2.5-VL-3B-F, hyperapi-parse-intent are open-source backbones (Qwen2.5-VL-Instruct) fine-tuned by Hyperbots via LoRA (rank 32, α 64; vision-encoder final layer + language decoder; lr 1e-4 vision / 5e-4 language; batch 1, grad-accum 10; 7 epochs 3B / 5 epochs 7B; 2× H100 80 GB, CUDA 12.3). Inference: top-p 0.9, temp 0.1, frozen 10-step layout prompt. Latency p50 2,548 ms / p99 3,319 ms, 3% fail rate; cost ≈ $0.19 / 1k successful calls (A100 40 GB @ ~$2/hr).

Limitations

Fine-tune-vs-zero-shot asymmetry; PaIRS PC-only; corpus-level aggregate only (no by-class/quality/field slices); parse-intent n unknown; single version (v1, 2026-05-18). See paper/SAVIOR-Bench.md §5.

Positioning & honest depth roadmap

Positioned vs public document-parsing benchmarks in paper/SAVIOR-Bench.md §3 (related-work matrix, incl. PureDocBench): SAVIOR-Bench's current distinctness is financial-document specificity + provenance-tagging + parse-intent coverage; it does not yet match PureDocBench-class evidence depth (regenerable source-traceable corpus, factory-reproduced cells, per-category/degradation slices). That gap is named, not faked — §10 "Path to PureDocBench-depth" is the explicit roadmap to close it (factory re-reproduction, slices, regenerable corpus, broader model coverage).
