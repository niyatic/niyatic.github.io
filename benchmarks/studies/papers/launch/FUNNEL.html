0. The strategy in one line

The credible, surprising-but-true finding (a modest fine-tune edge, honestly disclosed) distributed to the right technical communities is the entire growth lever. Manufactured virality is explicitly off the table — a trust spike that leaks is a net loss. Earned distribution over paid amplification (Frugality).

3. Funnel & UTM instrumentation spec

3.1 The honest funnel


Impressions (per channel, platform-native analytics — observed, not modeled)
   └─> Link clicks (UTM-tagged, server-side resolvable)
        └─> Landing/leaderboard/paper views (analytics pageview)
             └─> Signup-page reaches (UTM carried through)
                  └─> Signups completed (instrumented conversion event)
                       └─> Activated signups (made ≥1 real API call) [post-24h]

Every stage is a measured count. Impressions at the top are platform-reported (HN/X/LinkedIn native), explicitly labelled "platform- reported, not independently verified". No stage is inferred from another; if a stage can't be measured it is shown as n/a (not instrumented), never back-filled.

3.2 UTM convention

<SIGNUP_URL> base = the HyperAPI signup page. Every outbound link uses:


?utm_source={hn|x|linkedin|reddit|hf|blog|discord}
&utm_medium={social|community|referral|organic}
&utm_campaign=savior-bench-v1
&utm_content={post-a|post-b|post-c|blog-cta|leaderboard-cta|paper-cta}

Examples:

  • HN post link → utm_source=hn&utm_medium=social&utm_campaign=savior-bench-v1&utm_content=post-a
  • Blog body CTA → utm_source=blog&utm_medium=referral&utm_campaign=savior-bench-v1&utm_content=blog-cta
  • Leaderboard CTA → utm_source=leaderboard&utm_medium=referral&utm_campaign=savior-bench-v1&utm_content=leaderboard-cta
  • Paid (if ever human-gated) → utm_medium=paid (kept strictly separate)

UTMs must survive the redirect chain to the signup completion event (verify in the T-24h dry-run; a UTM that drops at signup = G4/G6 not green).

3.3 Dashboard (honest, real-time)

A single page, one row per channel, columns: impressions (platform-reported) | clicks | views | signup-page reaches | signups | activated (post-24h) | click→signup % | notes.

Mandatory dashboard rules:

  • A plain "vs target" line: `signups so far: N / 1000 aspirational

(T+Hh)` — shown even when N is far below 1000. "Below target" is stated in plain words, never hidden, smoothed, or re-based.

  • Bot/spam signups are flagged and excluded from the headline count

(and the exclusion count shown) — never padded.

  • No projection/extrapolation presented as a result. A projection, if

shown at all, is labelled "projection, not measured" and kept separate from actual counts.

  • Source of every number is captioned (platform analytics vs server-side

event).

3.4 What we do NOT do (enforced)

  • No bought signups / stars / followers / upvotes; no sock-puppets; no

coordinated vote rings; no incentivized fake signups.

  • No "1000 signups" claim unless 1000 real, non-bot, instrumented

signups actually occurred.

  • No screenshot of an inflated or cherry-picked dashboard.
  • Paid spend, if ever used, is human-gated and labelled paid everywhere.

4. The 24h iterate loop

Checkpoints at T+1h, T+3h, T+6h, T+12h, T+24h. At each, log to a running launch/24H-LOG.md (created at launch, not now):

  1. Read the dashboard (real numbers only).
  2. Diagnose the weakest funnel stage (e.g. clicks high, signups low →

landing/signup friction, not a copy problem; impressions low → channel/ timing problem).

  1. Allowed iterations (honest only):
  • Swap which honest sub-finding leads a follow-up post (e.g. lead with

the recall/layout-decouple finding instead of the headline gap) — same true numbers.

  • Fix landing/signup friction (a real UX issue is the highest-leverage,

fully honest lever).

  • Re-weight channel effort toward what's converting.
  • Answer questions substantively in-thread (earned engagement).
  1. Forbidden "iterations": changing a number to look better, adding

manufactured engagement, re-posting to game ranking, hiding a below-target line.

  1. Log the decision + the honest rationale + the next checkpoint.

At T+24h: write the honest retrospective — actual impressions→clicks→ signups by channel, what the artifact's real merit converted, whether the aspirational target was met (stated plainly either way), and what to harden next. This becomes input to the post-launch hardening path (§5).
