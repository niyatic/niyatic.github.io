3. "Very-hard enterprise table" criteria + curation pipeline

A table qualifies as very-hard enterprise if it meets ≥2 of:

  • borderless / partially-ruled rules (no full grid lines);
  • merged/spanning header or sub-total cells (rowspan/colspan > 1);
  • multi-page continuation with repeated headers;
  • photographed or low-quality scan (skew/noise/JPEG artifacts);
  • finance semantics: nested line items, currency/locale formatting,

negative-in-parentheses, footnote-linked figures;

  • multilingual or rotated content.

Pipeline (each stage logs counts only; no content leaves the curator):

  1. Inventory the doc-like subset (incl. controlled zip expansion

in a sandbox) — metadata + page count, no text extraction yet.

  1. Doc-class filter — keep invoices/POs/statements/financials;

drop non-enterprise PDFs (manuals, web saves, image-PDFs).

  1. Table-presence detection — run a table detector; keep pages with

≥1 table.

  1. Hardness scoring — apply the §3 criteria; bucket easy/medium/

very-hard.

  1. GT annotation — expert cell-grid + HTML annotation for the

very-hard bucket (the cost bottleneck); public corpora supply ready GT for anchors.

  1. PII / licensing review (data-privacy-compliance) — redact or

exclude; only license-clean items enter any released split; internal docs stay internal (metrics-only publication).

  1. Provenance manifest — per item: source, license, class, hardness,

GT method (expert vs corpus-native), split.

4b. Annotation rubric (very-hard enterprise slice) — DESIGNED

The cost bottleneck (§3 stage 5) is human cell-grid + HTML annotation. Rubric below makes it reproducible across annotators. EXECUTION-PENDING — no annotation performed in this environment (no annotators, no real docs).

Unit of annotation: one detected table region → one canonical HTML tree + the explicit grid cell list (the §1 wiring contract in plan/TABULAR-BENCHMARK-WIRING.md).

Per-cell required fields and decision rules:

  • Cell boundary. A cell is the smallest rectangle bounded by ruled

lines OR by consistent whitespace gutters. Borderless: gutter must be ≥1.5× median intra-word gap to split a column (logged threshold, not per-annotator taste).

  • Spanning. rowspan/colspan recorded as integers; a value

visually centered across N sub-columns under a group header → header colspan=N; never duplicate the value into N cells.

  • Header vs body (is_header). Header = label row/col that names a

dimension. Stub column (leftmost row labels) tagged is_header on the cell when it names the row's measure; ambiguous → body + flag header_ambiguous (excluded from TEDS-Struct strictness, kept in content scoring).

  • Cell text normalization. Transcribe verbatim glyphs incl.

currency symbols, thousands separators, and negatives. Negatives in parentheses → keep parentheses verbatim in text; a parallel numeric_value is NOT annotated (TR-000 is structure+text, not semantics — semantics belong to extract/validate tasks).

  • Empty vs missing. Visually empty cell that is structurally present

→ empty-string cell (counts toward grid). Cell absent from the grid (ragged table) → omitted, grid records the true span layout.

  • Multi-page continuation. Each physical page annotated separately

with meta.complexity=multipage + a continuation_group id; the metric scores per-page (the model also sees per-page input) — joining is a downstream task, not TR-000 GT.

  • Reading order. Row-major, top-to-bottom, left-to-right per visual

layout; rotated tables annotated in their de-rotated reading frame and tagged meta.complexity=rotated.

Adjudication ladder: every very-hard item is annotated by 2 independent annotators; disagreements go to a 3rd senior adjudicator whose call is the gold. All three versions retained for the IAA computation (§4c) and for an error-taxonomy audit.

4c. Inter-annotator agreement (IAA) protocol — DESIGNED

IAA is measured with the same metrics the benchmark scores models with (no separate ad-hoc agreement number — agreement and model quality are then on one comparable scale):

  • Structure agreement: pairwise GriTS-Top and TEDS-Struct

between annotator A and annotator B per item; report mean ± bootstrap ci95 (seed 20260516, same bootstrap_ci as the harness).

  • Content agreement: pairwise TEDS (full) and **cell-F1

(content)**.

  • Categorical fields (is_header, span integers): Cohen's κ

per field; flag κ < 0.6 fields for rubric tightening before scale-up.

  • Acceptance gate: a slice is admissible as GT only if median

pairwise GriTS-Top ≥ 0.95 and TEDS-full ≥ 0.90 on a ≥50-item calibration round; below that, rubric is revised and re-calibrated (do not ship GT the annotators themselves can't agree on — it caps the meaningful model ceiling).

  • Annotator-vs-gold ceiling: after adjudication, report

single-annotator-vs-gold TEDS as the human ceiling; published model numbers are always shown against this ceiling so a model "beating GT" is visibly a noise/ceiling artifact, not a claim.

  • Drift control: a fixed 30-item anchor set is re-annotated

every 500 items; anchor-set IAA tracked over time to catch annotator drift (logged, not asserted).

Honest status: the protocol is fully specified; zero items have been annotated (no real corpus / annotators in this environment). All IAA numbers are EXECUTION-PENDING and MUST NOT be reported until a real calibration round runs.
