DocumentAI Benchmark Factory — CTO Plan (PLAN → answers + human approval → go)

Status: PLAN by frontier-lab-cto. Team NOT recruited/instantiated until human approves + answers the blocking questions. Mirrors ABM/repo-factory gating.

1. Mission

A trusted leaderboard: 100+ models per task, across all external + internal Hyperbots datasets, for every finance-doc-AI task, operated like the HuggingFace Open LLM Leaderboard / DeepMind eval suites — reproducible, contamination-controlled, statistically honest, every cell drillable to a rerunnable command.

2. Team

frontier-lab-cto (lead) · benchmark-product-manager (HF/DeepMind-style) · eval-harness-engineer · dataset-curator · model-wrangler · contamination-auditor · stats-methodologist · leaderboard-frontend-eng · qa-reproducibility. (.claude/agents/)

3. Method

  • Task taxonomy: the finance-doc-AI task catalog (assumed = the 549 implemented + planned catalog in finance-doc-agents-hyperapi; confirm — blocking Q).
  • Datasets: registry of external/public sets + internal Hyperbots sets (access + license — blocking Q); fixed splits; contamination audit before any number ships.
  • Models: 100+ via adapters — open-weights (HF hub) + hosted APIs; pinned versions; per-model cost/throughput budget.
  • Harness: one fair, pinned harness; identical prompt/decoding per task family; metrics + CIs + n; every cell = a rerunnable command.
  • Leaderboard: public UI, filterable by task/dataset/model-family, per-cell drill-down; documented refresh cadence; anti-gaming + held-out policy.
  • Gate: qa-reproducibility independently re-runs sampled cells; CTO STOPs any irreproducible/contaminated result.

4. Scope decision (BLOCKING — asked)

100+ models × all tasks × all datasets of real inference is significant compute/cost. v1 options: (A) Full real-run now (needs compute + API budget + keys). (B) Rigorous harness + methodology + dataset/model registry + seed leaderboard (a real, defensible subset run end-to-end) with a pinned, documented path to scale to 100+ — scale as budget allows. Recommend (B) as v1 (credible + reproducible immediately; honest about coverage), scaling to (A).

5. Go sequence (on approval + answers)

  1. Recruit/instantiate the 9 agents + benchmark-factory skill.
  2. PM publishes leaderboard product spec + task/metric catalog + dataset registry + repro contract.
  3. Harness built + validated on a seed task×dataset×(small model set); contamination audit; QA re-run.
  4. Scale models/tasks/datasets per the approved scope toward 100+; leaderboard deployed; cadence live.
