SAVIOR-Bench is a 509-document, expert-annotated financial-document benchmark spanning five corpora (UCSF Industry Documents, in-house synthetic, misc. public financial, Inv3D, FUNSD) with inter-annotator agreement (IAA) of 0.761 (fuzzy string similarity over 187 double-annotated pairs). The benchmark defines a single parse task (DC-000) as a three-facet task — parse = extraction + layout awareness + bounding box — evaluated on identical inputs, prompts, and scoring:

  • Facet 1 — Extraction. Field/value/line-item extraction AND OCR

text-recognition accuracy: word-level multiset recall and precision, parse-intent F1 and word error rate (WER), and OCR normalized edit distance. Measured (attested) — the SAVIOR-Bench OCR table (21 systems), the parse-intent table (23 systems), and a 31-model OCR edit-distance leaderboard (externally-sourced; §5.1.4).

  • Facet 2 — Layout awareness. Reading order, layout-element structure,

table/region structure: layout-element accuracy, reading-order accuracy, and PaIRS (pairwise spatial relationship score, reported only for prompt-controlled models). Defined; not yet measured — see §2.3.

  • Facet 3 — Bounding box. Visual grounding: correctness of predicted

bounding boxes / field-to-region localization: bbox IoU and field-to-region grounding accuracy. Defined; not yet measured — see §2.3.

We report all seven HuggingFace-grade pillars: side-by-side comparison, dataset composition, accuracy slices (to the extent the attested data supports them), latency distributions, cost-per-call and cost-per-correct, failure modes, and methodology/reproducibility.

On parse-intent (23 systems), the fine-tuned hyperapi-parse-intent [FT] leads at F1 0.8328 / WER 0.2589; the best well-documented zero-shot system is GPT-4o (0.8052 / 0.4232) — a +0.0276 F1 fine-tune gap. (An under-documented panel row, DocIntentOCR-3B, posts a marginally higher 0.8102 / 0.5622 with no recorded vendor/checkpoint/config; reported as-attested, not used as the comparison anchor — see §5.1.1.) On SAVIOR-Bench OCR (21 systems), the fine-tuned Qwen2.5-VL-7B-F [FT] leads recall at 0.9294, narrowly above strong zero-shot cloud baselines Textract (0.9233) and Azure-DI (0.9202); on layout fidelity, the highest PaIRS scores go to zero-shot PaddleOCR-VL-1.5 (0.8766) and dots.mocr (0.8191) — but the SAVIOR-Bench panel is non-prompt-controlled (NPC), so PaIRS here is a format-compatibility proxy, not a directly comparable layout-quality ranking (see §2.2 / §8; the body retracts any PC-vs-NPC PaIRS contrast).

1. Introduction

Financial-document understanding (invoices, packing slips, forms, multilingual scans, low-quality PDFs, complex tables) places two demands on an OCR/VLM stack that generic OCR benchmarks under-measure: (1) whole-token fidelity — downstream LLM extraction needs Total, not Totl; a 1-char edit that char-level CER rewards still breaks a key match; and (2) layout fidelity — reading order and spatial structure determine whether a table parses. SAVIOR-Bench scores both, on a financial-document corpus, with a single frozen prompt and pinned decoding per task family, against a wide panel of vendor, frontier, and open-source systems.

2.2 The parse task is three facets

parse (DC-000) is one task with three facets: a parsed document is only correct if its text/fields are right (extraction), its structure is right (layout awareness), and each field is placed correctly on the page (bounding box). The three facets are scored independently, each with its own metric(s), and a system can lead one facet while losing another. The facet definitions and metrics are mirrored in harness/tasks.json DC-000 (facets).

Facet 1 — Extraction (MEASURED, attested)

Field/value/line-item extraction accuracy AND OCR text-recognition accuracy.

  • Word-level recall & precision (multiset). Reference and hypothesis are

treated as token multisets; recall = matched / reference tokens, precision = matched / hypothesis tokens. Word-level (not char-level) is deliberate: downstream LLM extraction consumes whole tokens, so Totl ≠ Total must be penalized fully, not partially.

  • Parse-intent F1 / WER. Structured-parse variant: F1 over extracted

intent fields (↑ better); WER over the transcription (↓ better).

  • OCR normalized edit distance (ocr_edit_distance). Normalized

Levenshtein edit distance of predicted vs gold transcription (range 0–1, ↓ better). Populated from the externally-sourced 31-model OCR edit-distance leaderboard (ocr-edit-distance-v1; §5.1.4).

Facet 2 — Layout awareness (DEFINED; NOT YET MEASURED)

Reading order, layout-element structure, table/region structure.

  • Layout-element accuracy (layout_element_accuracy). Accuracy of

predicted layout-element labels/structure (title, paragraph, table, figure, region) vs the gold layout. Scored as element-match F1 over the gold layout tree (range 0–1, ↑ better).

  • Reading-order accuracy (reading_order_accuracy). Correctness of the

predicted reading order of layout elements vs the gold sequence. Scored as a normalized rank-correlation / sequence-match over the ordered element list (range 0–1, ↑ better).

  • PaIRS (pairwise spatial relationship score). Quadratic spatial penalty

over pairwise token-position relationships, z-scored, for layout/reading- order fidelity. Reported only for prompt-controlled (PC) models — systems whose output layout can be steered by the frozen prompt. Non-prompt-controlled (NPC) systems (e.g. cloud document-AI APIs that emit a fixed schema) are scored on format compatibility, and their PaIRS values are present but not directly comparable to PC PaIRS. This is a stated methodological limit, not a hidden one (see §8).

Status. This facet is defined here but not yet measured: no layout-awareness ground-truth dataset (gold layout trees / reading-order sequences) is in scope for SAVIOR-Bench v1. Its result status is "Not yet measured — facet defined, awaiting data." No layout-awareness number is fabricated. (The attested SAVIOR-Bench OCR table does carry PaIRS values — those are reported in §5.1.2 as a layout proxy, with the PC/NPC comparability caveat; the full layout-awareness facet — element + reading- order accuracy on gold layout trees — remains unmeasured.)

Facet 3 — Bounding box / visual grounding (DEFINED; NOT YET MEASURED)

Correctness of predicted bounding boxes / field-to-region localization.

  • Bounding-box IoU (bbox_iou). Mean Intersection-over-Union of

predicted vs gold bounding boxes over all grounded fields/regions (range 0–1, ↑ better).

  • **Field-to-region grounding accuracy

(field_to_region_grounding_accuracy).** Fraction of extracted fields whose predicted box matches the gold region above an IoU threshold (e.g. IoU ≥ 0.5) (range 0–1, ↑ better).

Status. This facet is defined here but not yet measured: no bounding-box / visual-grounding ground-truth dataset (gold field boxes) is in scope for SAVIOR-Bench v1. Its result status is "Not yet measured — facet defined, awaiting data." No bounding-box number is fabricated.

6. Analysis

This section reads the §5 numbers; it introduces no new data.

  1. The fine-tune advantage is narrow and metric-specific, not universal.

On parse-intent, hyperapi-parse-intent [FT] leads the best well-documented zero-shot system by +0.0276 F1 (0.8328 vs GPT-4o 0.8052; +0.0226 vs the under-documented DocIntentOCR-3B 0.8102) with a large WER reduction. On OCR recall the fine-tuned Qwen2.5-VL-7B-F [FT] leads the best zero-shot system (Textract) by only +0.0061 (0.9294 vs 0.9233). On OCR precision and PaIRS, zero-shot systems lead (GPT-4o precision 0.9322; PaddleOCR-VL-1.5 PaIRS 0.8766). Fine-tuning buys a small recall/F1 edge, not a layout or precision edge.

  1. Recall and layout decouple. PaddleOCR-VL-1.5 is near the bottom on

recall (0.7220, rank 17) yet first on PaIRS (0.8766); Textract is second on recall (0.9233) but mid-pack on PaIRS (0.5117). High transcription recall does not imply faithful reading order — which is exactly why the benchmark scores both, and why PaIRS is reported separately.

  1. WER and F1 are not monotone on parse-intent. PaddleOCR-VL-1.5 reaches

F1 0.7260 (rank 5) with WER 2.9641; DeepSeek-OCR-2 reaches WER 12.3541 at F1 0.3430. Systems that hallucinate verbose output can score moderate field-F1 while transcription WER explodes — a pattern visible only because both metrics are reported.

  1. PC vs NPC is a comparability boundary, not a quality ranking. The

PaIRS leaders among PC models (Gemini-2.5-Flash 0.6074, Qwen2.5-VL-7B-Instr. 0.5629) are not directly comparable to NPC PaIRS proxies. The matrix in §3 marks our lack of per-track slices ✗; analysis here stays at the corpus-aggregate level the attested data supports and does not infer slice-level conclusions.

No claim in this section exceeds what the attested aggregate cells support; where a finer cut would be needed it is deferred to §10, not asserted.

7. Case Studies

Worked examples drawn only from the existing attested data and the failure modes documented in paper/SOURCE-EXTRACT-Benchmark-VLM-OCR.md (← the Hyperbots-provided Benchmark-VLM_OCR.pdf). These are honest readings of data already in the paper — no new measurements, no per-document scores (none exist in the attested source), no invented numbers. Each case states explicitly which numbers are attested and which are qualitative observations from the PDF's documented failure modes.

Case Study 1 — Fine-tune narrows but does not close the multilingual / very-dense cluster

Attested basis: Pillar 6 (PDF) lists "multilingual documents; ambiguous very-dense layouts" as the residual failure cluster for the fine-tuned SAVIOR-Train models themselves; the PDF states the cluster "persists even after fine-tuning." Observation: the fine-tuned OCR leader's recall edge over the best zero-shot system is only +0.0061 (0.9294 vs Textract 0.9233), consistent with fine-tuning improving the common case while a hard residual cluster caps the headroom. What is NOT claimed: no per-language or per-density accuracy number is given — the attested source has no such slice; this is a qualitative reading, flagged as such.

Case Study 2 — High recall, weak layout (transcription ≠ structure)

Attested basis: Textract recall 0.9233 (rank 2) with PaIRS 0.5117; PaddleOCR-VL-1.5 recall 0.7220 (rank 17) with PaIRS 0.8766 (rank 1). Observation: a system can recover almost all tokens yet emit them in a poor reading order (Textract), while another recovers fewer tokens but preserves spatial structure (PaddleOCR-VL-1.5). For downstream table parsing, PaIRS captures a failure mode that recall alone hides. What is NOT claimed: PaIRS comparability is PC-only; both rows here are NPC, so their PaIRS is a format-compatibility proxy (see §2.2 / §8) — the contrast is illustrative of the metric's purpose, not a ranked claim.

Case Study 3 — Verbose hallucination inflates WER while F1 survives

Attested basis: parse-intent — PaddleOCR-VL-1.5 F1 0.7260 / WER 2.9641; DeepSeek-OCR-2 F1 0.3430 / WER 12.3541; contrast GPT-4o F1 0.8052 / WER 0.4232. Observation: field-level F1 can remain moderate even when the transcription is heavily padded with hallucinated tokens (WER ≫ 1). Reporting WER alongside F1 exposes systems whose structured output looks acceptable but whose raw transcription is unreliable. What is NOT claimed: no per-document error inspection (the attested source is aggregate only).

Case Study 4 — Fine-tuning does not buy precision or layout

Attested basis: OCR precision leader is zero-shot GPT-4o (0.9322), ahead of both fine-tuned rows (Qwen2.5-VL-7B-F 0.9157, Qwen2.5-VL-3B-F 0.9101); PaIRS leaders are zero-shot (PaddleOCR-VL-1.5 0.8766, dots.mocr 0.8191) vs fine-tuned 0.7066 / 0.7068. Observation: the SAVIOR-Train objective improves recall/F1 but not precision or reading-order fidelity — a candid counter-result that the fine-tune-first framing must not obscure. What is NOT claimed: nothing beyond the attested aggregate cells.

8. Limitations

  1. Fine-tuned vs zero-shot (primary). hyperapi-parse-intent,

Qwen2.5-VL-7B-F, Qwen2.5-VL-3B-F are fine-tuned on SAVIOR-Train; every other row is zero-shot native. Cross-setting comparisons are not apples-to-apples. The fine-tune advantage observed is recall/F1-specific and modest (parse-intent +0.0276 F1 over best well-documented zero-shot GPT-4o; OCR recall +0.0061 over best zero-shot Textract; no fine-tune lead on OCR precision or PaIRS).

  1. PaIRS is PC-only. PaIRS is directly comparable only among prompt-

controlled models. NPC systems' PaIRS is a format-compatibility proxy and must not be ranked against PC PaIRS.

  1. Attested, not factory-reproduced. All numbers are in-house-run and

user-attested (2026-05-18), not re-derived under the deterministic factory harness. No CI is available (ci95=null) — intervals are "reported (no CI)", never fabricated. See the explicit status box in Appendix D.

  1. No accuracy slices. The attested source is corpus-level aggregate

only; by-class / by-quality / by-field accuracy is not reported because the underlying per-document scores are not in the attested data. The §3 matrix marks this ✗ and §10 names the work to close it.

  1. No degradation / robustness tracks. SAVIOR-Bench v1 has no

systematically degraded variants; robustness is not measured (✗ in §3).

  1. n asymmetry. OCR table is n=509; the parse-intent table has no n

in the attested source — reported as unknown, not assumed.

  1. Not a regenerable corpus. The corpus is a static attested set with

mixed-source licensing; it is not source-traceable / regenerable in the PureDocBench sense (✗ in §3). The fixture audit (Appendix B) covers our own ground truth only, not a third-party benchmark.

  1. Single benchmark version. SAVIOR-Bench v1 only; dated 2026-05-18.

Model versions are as named/dated in the tables.

9. Conclusion & Future Work

SAVIOR-Bench v1 is a provenance-tagged, expert-annotated financial-document benchmark that scores OCR transcription, layout fidelity, and parse-intent extraction on identical inputs with a frozen prompt and pinned decoding, across a broad named panel — with the fine-tune-vs-zero-shot distinction surfaced first. Its empirical findings are deliberately modest: fine-tuning buys a narrow, metric-specific recall/F1 edge, recall and layout decouple, and the multilingual / very-dense cluster is a hard residual.

The honest position of this paper is that it has format parity but not evidence-depth parity with the deepest public document-AI benchmarks. The related-work matrix (§3) marks every dimension we do not yet satisfy with ✗. Future work is not to imitate the shape of a deeper benchmark but to do the underlying evidence work named in §10 — factory re-reproduction, per-category / per-quality / per-field slices, degradation tracks, a regenerable source-traceable corpus, and broader per-model-documented coverage.
