3 · The seven HuggingFace-grade pillars

1. Vendor parity — side-by-side on identical inputs/prompts/scoring

~21 OCR + 23 parse-intent systems spanning cloud document-AI APIs, frontier multimodal LLMs, and open-source OCR/VLM stacks. One frozen prompt template per task family; model chat-formatting confined to adapters. Three rows are SAVIOR-Train fine-tunes (Hyperbots-IP); every other row is zero-shot vendor-native. [FT] labelling is inline and non-removable.

→ paper/SAVIOR-Bench.md §5.1

2. Datasets + document classes

509 expert-annotated financial documents across UCSF Industry Documents (46.95 %), in-house synthetic (17.88 %), misc. public financial (17.49 %), Inv3D (9.04 %), FUNSD (8.64 %). Classes: invoices, packing slips, forms, multilingual scans, low-quality PDFs, complex tables. IAA 0.761 over 187 double-annotated pairs. Mixed-source license — access on request.

→ paper/SAVIOR-Bench.md §2.1

3. Accuracy by field type AND document quality

Honest limit: the attested source is corpus-level aggregate only. The PDF gives the source-corpus mix that fixes the stratification the benchmark could support, but per-document / per-class / per-quality / per-field scores are not in the attested data. Producing one would be fabrication — deferred to a factory re-run (§10 R3).

→ paper/SAVIOR-Bench.md §5.3

4. Latency distributions

From the Hyperbots source-of-record PDF, SAVIOR-Train fine-tuned inference path: p50 2,548 ms, p99 3,319 ms, 3 % failure rate (KV-cache bound; 2-GPU mitigation noted). No p90/p999/full-histogram is asserted — the source provides only these three points.

→ SOURCE-EXTRACT §3

5. Cost-per-call AND cost-per-correct

A100 40 GB at ~$2/hr (infrastructure-derived, not a price quote): cost per attempted call $0.000186, cost per successful call $0.000191 (≈ $0.19 per 1,000 successful calls). The attempted-vs-successful gap reflects the 3 % failure rate.

→ SOURCE-EXTRACT §4

6. Failure modes (ours / competitors / residual)

Ours (FT): multilingual docs, ambiguous very-dense layouts. Competitors (ZS panel): vertical text, fine print, multi-column layouts, logo-embedded text, degraded scans, handwriting, structured headers, bold-styled text. Residual: multilingual + very-dense cluster persists even after fine-tuning. No per-class incidence count is asserted (Appendix C explicitly flags "no per-class counts available").

→ paper/SAVIOR-Bench.md §5.6 + Appendix C

7. Methodology · rubric · reproducibility

Word-level multiset recall + precision (whole-token, not char-level); PaIRS layout (PC-only; NPC = format-compat proxy); parse-intent F1 (↑) / WER (↓). Decoding pinned: temp 0.1, top-p 0.9, max_tokens 2048, seed 20260516 (harness/tasks.json DC-000). One frozen prompt per family. Reproduction status box (Appendix D.2): attested — not factory-reproduced. Factory harness, pinned env, repro_check are defined and in place but not yet run against these cells.

→ paper/SAVIOR-Bench.md §2.2 / §5.7 / Appendix D

+ Format calibration vs PureDocBench (honest depth audit)

Standalone reviewer note governing P2.5: SAVIOR-Bench matches PureDocBench's format discipline but does not match its evidence depth (no per-track/-category slices, no degradation tracks, no CI, not regenerable, attested-not-reproduced). The related-work matrix in §3 of the paper marks ✗ on every dimension not satisfied. §10 is the named gap roadmap (R1–R7).

6 · Methodology & reproducibility summary

Rubric (in code, not prose)

  • Word-level recall / precision — multiset over whole tokens. Char-level CER deliberately rejected; downstream LLM extraction consumes whole tokens (Totl ≠ Total → full penalty).
  • PaIRS — quadratic spatial penalty over pairwise token-position relationships, z-scored. Reported only for prompt-controlled (PC) models. NPC rows show PaIRS as a format-compatibility proxy, not a comparable ranking.
  • Parse-intent F1 / WER — structured-parse variant: F1 over extracted intent fields (↑); WER over the transcription (↓).
  • TR-000 (registered-not-yet-run): primary teds; secondary grits, cell_f1.

7 · Honest limitations (in-body, not footnoted)

  1. Fine-tuned vs zero-shot (primary). Three rows (hyperapi-parse-intent, Qwen2.5-VL-7B-F, Qwen2.5-VL-3B-F) are LoRA fine-tunes on SAVIOR-Train; every other row is zero-shot vendor-native. Cross-setting comparisons are not apples-to-apples. The fine-tune advantage is narrow, metric-specific, and modest: parse-intent +0.0276 F1 vs best well-documented zero-shot (GPT-4o); OCR recall +0.0061 vs Textract; no fine-tune lead on OCR precision or PaIRS.
  2. Attested, not factory-reproduced. All cells are in-house-run, user-attested 2026-05-18 — not re-derived under the deterministic factory harness. The harness, pinned env, and repro_check exist (harness/PINS.txt, harness/repro_check.py) but have not been run against these cells. Per the 2026-05-18 attestation factory reproduction is optional/strengthening, not a publication gate.
  3. No confidence intervals. ci95 = null; reported as "no CI". CIs require per-document scores that do not yet exist. No interval is fabricated.
  4. No per-class / per-quality / per-field slices. The attested source is corpus-level aggregate. The paper's §3 related-work matrix marks this dimension ✗; §10 R3 names the work to close it. Generating a slice today would be fabrication.
  5. PaIRS is PC-only. Layout fidelity is directly comparable only among prompt-controlled models. NPC PaIRS is a format-compatibility proxy, not a quality ranking.
  6. Tabular-reasoning honest ceiling. TR-000's very-hard-enterprise corpus target is ~3.5k–6.8k docs, not 10k. The 10k figure is explicitly NOT asserted anywhere in the program; the program's own TABULAR-INDEX.md states this.
  7. Format-parity, not evidence-depth-parity vs PureDocBench. SAVIOR-Bench matches PureDocBench on disclosure discipline (IAA reported, own-fixture audit, FT/ZS disclosure, frozen decoding) but is strictly behind on evidence depth (no per-category / -degradation slices, no regenerable corpus, attested-not-reproduced). The paper's §3 matrix marks ✗ on every unsatisfied dimension; §10 is the named roadmap.
  8. n asymmetry. OCR table is n = 509; parse-intent table n is not reported in the attested source — reported as unknown, not assumed.
  9. Single benchmark version. SAVIOR-Bench v1 only, dated 2026-05-18. Model versions are as named/dated in the tables — competitors improve; the comparison is timestamped, not eternal.
  10. No degradation / robustness tracks. No systematically degraded variants; robustness is not measured (§3 ✗, §10 R4).
  11. Not a regenerable corpus. Mixed-source licensing; the corpus is static and attested, not source-traceable / regenerable in the PureDocBench sense (§3 ✗, §10 R5). The Appendix B fixture audit covers our own ground truth only — never a third-party benchmark audit.
