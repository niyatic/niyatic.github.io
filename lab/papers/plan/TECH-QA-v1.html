System summary (plain English)

A frontier-lab-cto-led, HF-Open-LLM-Leaderboard-style finance-document-AI benchmark factory. A scientific-method skill (SKILL.md) is the constitution: every leaderboard cell must be measured, version-pinned, and rerunnable by a stranger. v1 ships a stdlib-only, offline, deterministic harness (harness/run.py) that runs one pure predict(prompt,decoding)->text adapter on one task × one dataset, scores it with a per-family canonical metric (harness/metrics.py), computes a 2000-resample seeded bootstrap 95% CI + n, and writes a results/*.json carrying the literal rerun command + pins. A SEED leaderboard was run end-to-end: 4 tasks (DE-002/DE-003/DV-001/DC-001) × 1 clearly-labeled synthetic fixture × 3 deterministic adapters = 12 measured cells, plus a contamination audit, a repro_check.py STOP gate, and a static HTML leaderboard. External datasets (SROIE, CORD) are registered but download-gated; the internal-Hyperbots dataset adapter is an inert stub. A SCALE-TO-100.md documents the (not-yet-run) path to 100+ real models.

Per-area findings

1. Reproducibility — PASS (with one stale-pin caveat)

  • All 12 cells reproduce metric/CI/n bit-for-bit from their stored

rerun_command on the pinned commit + same Python. Fixture regenerates to the exact frozen sha256. Determinism verified across repeated runs.

  • repro_check.py runs and STOPs correctly on injected mismatch (exit 2).
  • CAVEAT (D2): every committed results/*.json has

pins.git_commit = 65aa750 (the scaffold commit) while the repo head is 4fe2440. The SEED numbers were generated at the prior commit and committed stale; they were NOT regenerated at the commit that ships them. They do still reproduce (harness logic unchanged between the two commits), so this is an integrity/process defect, not a numeric one — but the repro contract says "re-running on the pinned commit reproduces", and the pinned commit recorded in the artifact is not the commit a cloner is on.

2. Correctness — FAIL (one real metric bug)

  • bootstrap CI, field_f1, exact_match, numeric_tolerance: math verified by

independent re-derivation of 3 cells incl. CIs — exact match. n honestly reported (60). Tie-band = CI-overlap, statistically honest.

  • D3 (critical): bool_match is wrong for the negative class. It tests

substring containment against {"true","yes","valid","1"}; "valid" is a substring of "invalid", so a correct "invalid" prediction is parsed as True. Verified: bool_match("invalid",False)=0.0 (should be 1.0) and bool_match("invalid",True)=1.0 (should be 0.0). Effect: the entire DV-001 column is meaningless. The two adapters that actually compute validity score 0.8 (12 correct "invalid" answers marked wrong) and the always-"valid" rule-baseline also scores 0.8 — they only look tied because the bug exactly cancels for the constant baseline. This is a measured-but-incorrect cell that ships in the SEED leaderboard.

3. Honesty — PASS (strong)

  • SEED uses an explicitly labeled synthetic fixture; generator and registry

shout "NOT a real-world benchmark"; leaderboard has a persistent banner.

  • No fabricated SROIE/CORD contents anywhere; both loaders raise rather than

emit fake rows; registry records real source_url/mirror/license/splits and marks them requires-manual-download, sha256 PENDING.

  • Internal adapter genuinely inert: is_active() gated on two env vars; both

code paths raise NotImplementedError; cannot silently fabricate. Verified.

  • contamination/audit: synthetic = real structural argument with the live

sha256; external = honest REQUIRES-CHECK / PENDING-on-local-corpus, not a fake pass. Not a stub.

4. Robustness — PASS (with D1)

  • Fully offline (no network imports/calls anywhere). Deterministic

(pinned seeds for data, decoding, bootstrap, repro-sample). No float-nondeterminism (integer-indexed bootstrap, rounded to 6 dp).

  • Clean failure on unknown model/dataset/task and on missing external data.
  • D1 (high): contamination/audit.py crashes out-of-the-box. It globs

results/.json and the repo ships* results/_repro_check.json (no dataset key) -> KeyError. The README's documented order (audit.py then repro_check.py) only works on a pristine first run where _repro_check.json is absent; on the committed repo, and on every re-run, python contamination/audit.py fails. leaderboard/build.py correctly filters _-prefixed files; audit.py does not.

  • D4 (low): repro_check.py weak isolation. Its comment says "re-run into a

temp results dir to avoid clobbering, then compare", but it does not: it re-reads the same file run.py just overwrote and compares to the pre-read copy. It still catches a true mismatch (fault-injection confirmed), but it conflates "rerun is deterministic" with "rerun matches the committed artifact" and cannot detect a committed-artifact-vs-code drift (this is exactly how D2's stale commit slipped through the gate).

5. Scale claim — CREDIBLE (concrete, honestly bounded)

SCALE-TO-100.md is specific, not hand-wave: harness is genuinely task/dataset/model-agnostic (verified by reading run.py — adding a model = registry+adapter entry, no harness change); the one new component (HF-hub pinned-revision adapter) is sketched against the fixed predict contract; scale-up deps are pre-staged+commented in PINS.txt (transformers==4.44.2 etc.); cost/throughput fields exist per model in models.json; sharding/tiering/budget-gate and the release pipeline are named. Gaps: no executed proof of even one real HF model (the "only new code to scale" is unexercised); the 549-task / 100-model claim is a procedure, not a demonstration. Credible as a plan; unproven as an implementation.

6. Guardrails — PASS

Metrics computed in code, never asserted. HyperAPI accuracy appears only as "per apis.hyperbots.com" (leaderboard banner + adapter/registry notes), never injected into a measured cell. Hyperbots-IP rows factually framed ("open-source model fine-tuned by Hyperbots"), not disparagement of others. Zero HITL references. Every design doc carries ## Sources + ## Open assumptions. What changed

  • D3 (CRITICAL) — harness/metrics.py bool_match. Replaced substring

containment (any(w in pn for w in t)) with exact normalized-token equality (pn in t / pn in f) after _norm (strip+lower+ws-collapse). Hand re-derivation, re-confirmed in code: bool_match("invalid",False)=1.0, bool_match("invalid",True)=0.0, bool_match("valid",True)=1.0, bool_match("valid",False)=0.0, bool_match(" VALID ",True)=1.0. "invalid" no longer matches the positive set; normalization preserved.

  • D1 (HIGH) — contamination/audit.py. Glob now skips

os.path.basename(f).startswith("_") (mirrors leaderboard/build.py) and defensively skips rows lacking dataset/model. Works pristine and on every re-run; _repro_check.json no longer triggers KeyError:'dataset'.

  • D2 (HIGH) — truthful pins. harness/run.py git_commit() already

stamps live git rev-parse HEAD at run time; the defect was stale committed artifacts. SEED regenerated at the shipping commit so every results/.json pins.git_commit now equals repo HEAD (the immutable code commit cf2d9a5…, not the scaffold 65aa750…). Shipped as a two-commit split: commit 1 freezes the fixed harness code; commit 2 carries SEED artifacts regenerated against commit 1, so every results/.json pins.git_commit truthfully equals a real, immutable ancestor that contains the exact harness that produced the numbers. (A single commit cannot self-reference its own SHA inside committed artifacts; pinning the producing-code ancestor is the only structurally-honest state.)

  • D4 (LOW) — harness/repro_check.py isolation + drift-sensitivity.

run.py honors BENCH_RESULTS_DIR; repro_check now reruns into a tempfile.mkdtemp (auto-removed) and compares the fresh rerun vs. the stored committed artifact (no clobber). For pin drift it requires the artifact's pins.git_commit to be (a) a real commit, (b) an ancestor of HEAD, and (c) byte-identical in harness/ + fixture code to HEAD — i.e. the stored numbers are a faithful rerun of the current code. This catches the exact D2 failure (pinning the scaffold 65aa750, whose metrics.py differs) and bogus/unknown pins, while not imposing the impossible "pin == own commit" on committed artifacts. Rerun failure (returncode/missing file) and numeric/CI/n mismatch still STOP.
