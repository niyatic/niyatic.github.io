LEADERBOARD-SPEC — HyperAPI DocumentAI Benchmark (PM product spec)

Owner: benchmark-product-manager. Reviewed by frontier-lab-cto. Modeled on the HuggingFace Open LLM Leaderboard and Google DeepMind eval programs. Governing method: .claude/skills/benchmark-factory/SKILL.md.

1. Product goal

A leaderboard people trust: 100+ models × every finance-doc-AI task × all registered datasets, where every cell is measured, pinned, and rerunnable by a stranger. No self-reported numbers; no unpinned "vibes".

2. Task taxonomy + canonical metric

Source of truth: finance-doc-agents-hyperapi/docs/task-catalog-full.csv (549 implemented tasks across 30 patterns; 61,104 planned). Tasks are grouped into task families and each family declares ONE canonical metric, defined in code in harness/metrics.py (never prose):

FamilyExample task IDsCanonical metric
extractDE-002 line items, DE-003 payment termsfield-level F1 / exact-match per field
validateDV-001 totals validboolean accuracy
classifydocument-type classificationexact-match accuracy
calculatetax/total computationnumeric-tolerance accuracy (tol 1%)
reconcile / match / check(planned families)F1 / accuracy, defined in code when seeded

SEED tasks shipped: DE-002, DE-003, DV-001, DC-001 (representative of extract / validate / classify families; all map to implemented catalog tasks).

3. Model-submission / repro contract

  1. Submitter provides: a canonical model ID + a pinned ref (HF revision/

commit hash, API model+version date, or local file sha256) + an adapter (predict(prompt, decoding) -> text, pure, no task logic).

  1. The factory runs the harness; the submitter never self-reports numbers.
  2. Decoding is fixed per task family and identical across all models in it.
  3. Every cell stores its literal rerun command + full pins. Re-running on the

pinned commit must reproduce the metric within Monte-Carlo CI.

  1. qa-reproducibility re-runs ≥1 sampled cell per release; mismatch ⇒ STOP.

4. Dataset registry schema

datasets/registry/registry.json. Per dataset: id, name, type (external-public | synthetic-fixture | internal-hyperbots), source_url, license, public_availability_date, tasks[], split, loader, content_sha256, status. Splits frozen at registration. Internal-Hyperbots data enters ONLY through the pluggable adapter (datasets/loaders/internal_adapter.py) when a real path + creds are supplied; never fabricated. v1 = external/public-only + the inert internal adapter + a clearly-labeled synthetic fixture for harness validation.

5. Public leaderboard UX

Single static HTML (leaderboard/index.html, no build, no network):

  • Filter by task, dataset, model family.
  • Each cell shows metric, 95% CI, n, tie-band membership, and a

contamination flag.

  • Per-cell drill-down reveals the exact rerun command + pins + the

contamination audit record.

  • Hyperbots-IP rows are marked "Hyperbots IP" (open-source model fine-tuned by

Hyperbots); model IDs are factual entries, not competitive disparagement.

  • A persistent banner states SEED-scope honesty (synthetic fixture ≠ ranking).

6. Refresh cadence

  • SEED: on every harness/commit change (regenerated from results/).
  • Production target: monthly model-registry refresh + on-demand for new model

submissions; dataset registry frozen between named versions.

7. Trust / anti-gaming / contamination policy

  • No training/tuning on eval splits; dev/test separated by the registry.
  • Prompt templates public + frozen per family; no per-model prompt engineering.
  • Decoding fixed per family; no per-model decoding search.
  • Contamination audit per (model,dataset) before publish: source, license,

public-availability date vs. model cutoff, n-gram/canary overlap signal; unverifiable cases flagged, never silently dropped.

  • Outliers trigger an audit, not a silent rerun-until-good.
  • Metrics measured, never asserted; HyperAPI's own accuracy referenced only

"per apis.hyperbots.com" and never injected into a measured cell.

8. "What makes an entry valid" gate

Pinned model ref + adapter + frozen dataset split + canonical metric in code + contamination record + a rerun command that qa-reproducibility confirms matches within CI. Miss any ⇒ entry rejected by frontier-lab-cto.

loaders activate on local presence — no fabricated fallback.

  1. Internal-Hyperbots datasets: location/access/license human-provided later;

adapter stays inert until then.

  1. SEED cells use the synthetic fixture and are explicitly not ranking claims.
