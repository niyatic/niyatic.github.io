TABULAR REASONING — Literature Survey (enterprise document tables)

_Owner: cto-org chief-research-scientist + document-ai-research-scientist. Scope: table recognition / understanding in enterprise documents, the metric landscape, and the SOTA picture as of the assistant knowledge cutoff (Jan 2026). This is a design/selection survey for the benchmark factory. No model was trained or hosted in this environment._

0. Why tables are the hard core of enterprise doc AI

Enterprise documents (invoices, POs, packing slips, bank/credit-card statements, financial statements, 10-K/10-Q tables, insurance schedules, remittance advices) are dominated by tabular content: multi-line items, spanning headers, nested sub-totals, rotated/merged cells, borderless rules, multi-page tables with repeated headers, footnote-linked figures, and currency/locale formatting. General OCR + reading-order parsing (the DC-000 "parse" task) is necessary but not sufficient: a transcription can have high word recall yet still get the table structure wrong (wrong row/column assignment), which destroys downstream extraction and validation. Tabular reasoning is therefore tracked as a distinct task family from parse.

2. Key methods (lineage, what to know)

2.1 Detection / TSR specialist models

  • Table Transformer (TATR) + PubTables-1M (Smock et al., CVPR 2022) —

DETR-based table detection and structure recognition trained on the ~1M-table PubTables-1M corpus; introduced GriTS and the canonical open TSR baseline. Strong on scientific/financial PDF-rendered tables; weaker on photographed/low-quality scans and heavily merged cells.

  • Earlier TSR lineage: DeepDeSRT, TableNet, SPLERGE (split-merge),

GraphTSR / TGRNet (graph formulation of cell adjacency), LGPMA (mask alignment), and the TableMaster / EDD / TableFormer family that emits HTML structure tokens (the de-facto PubTabNet output convention).

  • Cascade/heuristic engines: PaddleOCR PP-Structure (layout +

TSR + cell OCR pipeline), Camelot/Tabula (rule-based, lattice vs stream — strong only on ruled digital PDFs; fails on borderless/scans).

2.2 OCR-2.0 / end-to-end transcription with structure

  • GOT-OCR2.0 — unified end-to-end model that emits structured formats

(incl. tables as HTML/Markdown/LaTeX) directly from the image.

  • Nougat (Meta) — academic-PDF → Markdown incl. tables; brittle off-

domain (math-paper bias) but a useful structure-emitting baseline.

  • MinerU / PDF-Extract-Kit — pipeline (layout + formula + table) that

wraps specialist TSR; popular open enterprise-PDF extractor.

2.3 Prompted VLMs (the program's chosen SOTA-via-open-prompted route)

  • Qwen2.5-VL (3B/7B/32B/72B) — strong document/table grounding,

emits HTML/Markdown tables and bbox-grounded cells; current open front-runner on doc tasks.

  • InternVL 2.5 / 3 — competitive doc-VLM family; OCR/table prompting.
  • dots.ocr, PaddleOCR-VL, olmOCR, DocOwl/UReader,

MiniCPM-V, Phi-3.5/Phi-4-vision — VLM-prompted table emitters spanning the small/efficient end.

  • Closed frontier (scientific-comparison only): GPT-4o/4.1, Gemini

1.5/2.x, Claude 3.x — used as anchors, not products.

The program thesis: a prompted open VLM (no fine-tune) can reach or beat specialist TSR on enterprise tables when scored with structure-aware metrics — to be measured, not asserted.

3. Metrics — choices and rationale

  • TEDS (Tree-Edit-Distance-based Similarity) — represents a table as an

HTML tree; similarity = 1 − normalized tree-edit distance over structure and cell text. TEDS-Struct ignores cell text (pure topology). PubTabNet standard. Sensitive to small text errors → pair with Struct.

  • GriTS (Grid Table Similarity) — from PubTables-1M; matches predicted

vs GT cell grids via 2D-LCS, reported as GriTS-Top (topology), GriTS-Con (content), GriTS-Loc (location). More robust to non-tree spanning structures than TEDS; chosen as the primary TR-000 metric.

  • Cell F1 / adjacency-relation F1 — ICDAR-2013/2019 style; precision/

recall over correctly recovered cells or neighbor relations. Chosen as the secondary TR-000 metric (interpretable, decomposable by error type).

  • Avoid single-number "accuracy"; report **per cell-type / per document-

quality / per table-complexity** slices, mirroring the SAVIOR-Bench stratification discipline already in this repo.

TR-000 decision: primary teds (with TEDS-Struct companion), secondary grits + cell_f1. (tasks.json registers teds/grits primary-secondary with cell_f1 per the mandate; both structure metrics are first-class — naming follows the directive's teds/grits ordering.)

3.1 Precise metric definitions (so the impl is unambiguous)

These are the formal definitions the harness/metrics.py impl (plan/TABULAR-BENCHMARK-WIRING.md §2) must satisfy. Stated from the published literature; not benchmarked here.

TEDS (Tree-Edit-Distance-based Similarity), Zhong et al. Represent each table as an HTML DOM tree T: nodes are <thead>/<tbody>/ <tr>/<td>/<th>, each cell node carries (colspan, rowspan, content). Compute the tree-edit distance EditDist(T_pred, T_gold) (Zhang– Shasha style) where node-substitution cost = 0 if tags match else 1, plus for matched cell nodes a content cost = normalized string edit distance of cell text in [0,1]. Then TEDS = 1 − EditDist(T_pred,T_gold) / max(|T_pred|, |T_gold|), |T| = node count. TEDS-Struct sets the content cost to 0 (pure topology). Range [0,1], higher better. Property: sensitive to small text errors → always report alongside TEDS-Struct.

GriTS (Grid Table Similarity), Smock et al. (PubTables-1M). Represent a table as a 2D cell grid (matrix where a spanning cell occupies multiple grid positions). For predicted grid Gp and gold Gg, find the maximum-similarity 2D alignment (a 2D extension of LCS / factored bipartite matching) of cells, then GriTS = 2·Σ f(c_p, c_g) / (|Gp| + |Gg|) (an F-measure form) over matched cells, where f is the per-cell agreement function:

  • GriTS-Top — f = 1 iff matched cells have identical

(rowspan, colspan) topology (structure only);

  • GriTS-Con — f = normalized text similarity of cell content;
  • GriTS-Loc — f based on IoU of cell bounding boxes (geometry).

Range [0,1], higher better. Advantage over TEDS: operates on the grid not a tree, so it handles non-tree spanning structures more robustly — why it is the co-equal structure metric for TR-000.

Cell-F1 / adjacency-relation F1 (ICDAR-2013 / ICDAR-2019 cTDaR). Two variants:

  • Cell-content F1: a predicted cell is a true positive iff its text

(normalized) AND its grid span match a gold cell; P = TP/(TP+FP), R = TP/(TP+FN), F1 = 2PR/(P+R).

  • Adjacency-relation F1 (ICDAR-2013 classic): build the set of

horizontally/vertically adjacent cell pairs; a relation is a TP iff both endpoints' content match and the adjacency direction matches. F1 over the relation sets. Decomposable by error type (split/merge/ shift), which is why it is the interpretable secondary metric.

All three: same lenient HTML→canonical normalizer applied to pred and gold identically (whitespace collapse, attr strip except span, tag-lowercase) so the metric measures structure, not formatting noise.

3.2 Reference algorithm spec (engineer-implementable)

§3.1 gives the definitions; this section is the algorithm an engineer implements directly into harness/metrics.py (WIRING §2) without re-deriving from papers. Pseudocode is deterministic and language-neutral; complexity bounds noted so cost is predictable on the TR-000 slice. Nothing here is executed or benchmarked — spec only.

(A) Deterministic table normalizer _normalize_table_html(html) applied identically to pred and gold:


parse html leniently (html5-tolerant; never raise on malformed pred)
keep only: table, thead, tbody, tr, td, th
for each cell:
    rowspan = int(attr "rowspan" or 1); clamp >=1
    colspan = int(attr "colspan" or 1); clamp >=1
    text    = collapse_ws(strip_tags(inner))          # \s+ -> " ", trim
    text    = unicode_NFKC(text); casefold OFF (case is signal)
drop attributes except rowspan/colspan
if no thead/tbody: wrap all rows in tbody (canonical)
drop fully-empty trailing rows and trailing columns (both sides equally)
return canonical tree  (also expose .to_grid() — see (C))

Edge cases (must be explicit, else pred/gold diverge silently): unparseable pred → return a 1-node empty-table tree (TEDS will score near 0, never crash); ragged rows → right-pad with empty cells to the max effective column count; overlapping spans → first-writer-wins in grid raster order, log a malformed_span flag to the item sidecar.

(B) TEDS via Zhang–Shasha tree edit distance. Nodes carry (tag, rowspan, colspan, text). Costs:


insert cost  = 1     delete cost = 1
substitute(a,b):
    if a.tag != b.tag:                 return 1
    if a.tag not in {td,th}:           return 0          # structural
    struct_cost  = 0 if (a.rowspan,a.colspan)==(b.rowspan,b.colspan) else 1
    if struct_only:  content_cost = 0
    else:            content_cost = norm_levenshtein(a.text, b.text) in [0,1]
    return 0.5*struct_cost + 0.5*content_cost            # cell node in [0,1]

Run classic Zhang–Shasha (postorder + LR-keyroots + forest-distance DP) to get EditDist. Then TEDS = 1 - EditDist / max(|T_pred|, |T_gold|), clamp to [0,1]; if both trees empty → define TEDS = 1.0. TEDS_Struct = same call with struct_only=True. Complexity O(|Tp|·|Tg|·min(dp,lp)·min(dg,lg)) (d=depth,l=#leaves) — fine for single tables (hundreds of nodes); add a 5 s/table watchdog → on timeout record teds=None + metric_timeout flag, never a fake 0.

(C) GriTS via factored 2D matching. .to_grid() rasterizes the canonical tree into a 2D array where a span occupies every covered (r,c) and each logical cell gets a stable cell_id. Exact GriTS maximizes Σ similarity over the 2D cell alignment; the published tractable factorization (Smock et al.) is: solve the row assignment and column assignment as two 1-D optimal sequence alignments (DP, gap cost 0), then within the induced cell correspondence solve a bipartite max-weight matching (Hungarian / scipy.optimize.linear_sum_assignment) on the per-cell similarity f:


f_topology(p,g) = 1 if (p.rowspan,p.colspan)==(g.rowspan,g.colspan) else 0
f_content (p,g) = 1 - norm_levenshtein(p.text, g.text)
f_location(p,g) = IoU(p.bbox, g.bbox)        # bbox required for Loc only
GriTS = 2 * sum(f over matched) / (|Gp| + |Gg|)        # F-measure form

unmatched cells contribute 0; empty/empty → 1.0. GriTS-Loc is skipped (reported null) when GT lacks bbox (HTML-only slices) — never substitute a content score for it. Cost: alignment O(R·C), Hungarian O(n³) in #cells — bounded per table.

(D) Cell-F1 / adjacency-F1. Content mode: a pred cell is TP iff some gold cell has equal normalized text and equal (rowspan, colspan) at the aligned grid anchor (use the (C) correspondence — do not re-match independently, keeps the three metrics consistent). Adjacency mode: from the grid build directed adjacency pairs (cell, right_neighbor) and (cell, down_neighbor) skipping spanned continuations; a relation is TP iff both endpoint texts match and the direction matches. P=TP/(TP+FP), R=TP/(TP+FN), F1=2PR/(P+R) (define F1=0 when P+R=0). Return {precision,recall,f1}; F1 is the leaderboard scalar, P/R retained for split/merge/shift error slicing.

Determinism contract: no RNG anywhere in metrics; ties in Hungarian broken by ascending cell_id; all three metrics consume the same normalized structure so a structural fix moves TEDS/GriTS/cell-F1 coherently. Golden unit fixtures (perfect match→1.0; one merged cell; one split cell; transposed header; empty table; malformed-HTML pred) to be added under harness/tests/ at impl time — EXECUTION-PENDING, not written or run here.

5. SOTA landscape (honest framing)

  • Specialist TSR (TATR-family) reports high TEDS/GriTS **on the PDF-

rendered scientific distribution it was trained on**; degrades on photographed/low-quality enterprise scans and on heavily merged finance tables.

  • Prompted frontier + strong open VLMs (Qwen2.5-VL class) have closed

much of the gap on table-to-HTML/Markdown and increasingly lead on messy real-world inputs in recent reports — but published numbers mix zero-shot vs fine-tuned and differ in TEDS variant; not directly comparable without a pinned harness.

  • Therefore: no SOTA cell is asserted here. The factory's job is to

produce reproduced-or-provenance-tagged TR-000 numbers under one pinned prompt + decoding, with fine-tune-vs-zero-shot disclosed — the same honesty contract as DC-000.
