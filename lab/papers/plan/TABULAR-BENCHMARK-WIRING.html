TABULAR REASONING — Benchmark Wiring DESIGN (TR-000 → harness/)

_Owner: cto-org benchmark-format-steward + document-ai-research-scientist. Scope: the interface spec by which TR-000 (family: tabular, primary teds, secondary grits/cell_f1) plugs into the existing pinned harness. Design only — every metric impl and dataset wiring below is EXECUTION-PENDING. No metric was run, no result cell exists. This doc makes the contract precise so a cto-org engineer can implement it without re-deriving the harness shape._

0. What already exists in harness/ (verified, not assumed)

Read from the live tree at master 0698f88:

  • harness/tasks.json — tasks[TR-000] is registered with frozen

prompt, decoding (temp 0.0, top_p 1.0, max_tokens 4096, seed 20260516), metric: "teds", metrics_secondary: ["grits","cell_f1"], _status: REGISTERED-NOT-YET-RUN.

  • harness/run.py — driver. Resolves tasks[a.task], LOADERS[a.dataset]

→ (loader_mod, dataset_kind), REGISTRY[a.model]. For each row: prompt = tdef["prompt"].format(input_text=r["input_text"]), out = predict(prompt, tdef["decoding"]), scores.append(M.score_item(a.task, out, r["gold"][a.task])). Then bootstrap_ci(scores, seed, resamples=2000) → (point, lo, hi).

  • harness/metrics.py — score_item(task_id, pred_text, gold) is a

per-task dispatch returning a float in [0,1]; raises KeyError for unknown task. Helpers: _norm, exact_match, field_f1, etc.

  • harness/repro_check.py — re-runs a stored result cell in a temp dir,

checks pin-drift against HARNESS_PATHS and the stored git commit.

  • harness/run.py LOADERS dict maps `dataset_id → (loader_module,

dataset_kind); loader modules live in datasets/loaders/ and expose load() returning rows with input_text and gold[task_id]`.

The wiring task is therefore four additive changes, each isolated and non-breaking to existing tasks (DC-000/DE-/DV-/DC-001 untouched).

# harness/metrics.py  additions  (EXECUTION-PENDING)

def _normalize_table_html(html: str) -> "ETree":
    """Lenient parse → canonical tree: lowercase tags, drop style/attrs
    except rowspan/colspan, collapse whitespace in cell text, strip
    empty trailing rows/cols. Same normalizer for pred AND gold so the
    metric measures structure, not formatting noise. Returns a tree
    object consumed by teds()/grits()."""

def teds(pred_html: str, gold_html: str, struct_only: bool = False) -> float:
    """Tree-Edit-Distance Similarity (Zhong et al., PubTabNet).
    1 - (edit_distance(T_pred, T_gold) / max(|T_pred|, |T_gold|)),
    cell-text substitution cost = normalized string edit distance
    (0 when struct_only=True → TEDS-Struct companion). Returns [0,1]."""

def grits(pred_grid: list, gold_grid: list, mode: str = "topology") -> float:
    """Grid Table Similarity (Smock et al., PubTables-1M). 2D-LCS match
    of predicted vs GT cell grids. mode ∈ {topology, content, location}
    → GriTS-Top / GriTS-Con / GriTS-Loc. Operates on the `grid` field
    (no HTML re-parse). Returns [0,1]."""

def cell_f1(pred_grid: list, gold_grid: list,
            match: str = "content") -> dict:
    """ICDAR-2013/2019-style precision/recall/F1 over recovered cells
    (match='content') or neighbor adjacency relations (match='adjacency').
    Returns {'precision':p,'recall':r,'f1':f1} — f1 is the scalar fed to
    the leaderboard; p/r retained for error-type slicing."""

3. repro_check contract for TR-000

repro_check.py re-runs a sampled stored cell and compares. For TR-000 the contract is:

  1. Determinism precondition: decoding.temperature == 0.0 and a

fixed seed (20260516) are already pinned in tasks.json → greedy decode ⇒ a faithful local/open model re-run must reproduce the stored point metric within the existing repro tolerance band. Hosted-API cells are provenance-tagged, not bit-reproducible (provider may change weights) — repro_check must skip API-kind cells the same way it treats other external-API provenance, asserting only provider/endpoint/model_id/run_date are present.

  1. Pin set: add datasets/loaders/tabular_loader.py and the slice

manifests to HARNESS_PATHS so a change to the metric impl, loader, or frozen slice triggers pin-drift and invalidates stored TR-000 cells (forces re-run before re-publish).

  1. Metric-availability guard: repro_check must treat a

NotImplementedError from score_item("TR-000",…) as "cell cannot exist yet" → any TR-000 row in results while the metric raises is a hard QA failure (defends "registered-not-yet-run").

  1. CI rule (inherited, unchanged): ci95 populated only from

bootstrap_ci over real per-item scores; null otherwise. No interval is ever hand-written for TR-000.

4. Stratified reporting (mirrors SAVIOR-Bench discipline)

Single-number TEDS is banned for headline claims. TR-000 results are reported sliced by the gold.meta keys from §1:

  • by source (enterprise-curated vs each public anchor — legibility),
  • by complexity (simple/spanning/nested/borderless/multipage/rotated),
  • by quality (digital/scan-clean/scan-degraded/photo).

Each slice carries n, point, and bootstrap ci95 (or null if n too small to bootstrap honestly). Leaderboard cell schema reuses the existing provenance fields + fine_tune_vs_zero_shot disclosure (mandatory) + teds_variant (full vs struct) so external numbers are not silently conflated (the §5.1.3 hazard from the SAVIOR paper).

5. Model adapter hook (cross-ref, not duplicated)

run.py calls predict(prompt, decoding) via REGISTRY[model]. For VLM table models the adapter (in harness/adapters/) must, for family == "tabular", substitute the image artifact for the text {input_text} while keeping the frozen prompt string as the instruction — identical instruction for every model in the family (the pinned-prompt contract). Concrete per-model adapter behavior (prompt realization, decoding reconciliation, expected invocation) is now specified in plan/TABULAR-MODELS-TO-BENCHMARK.md §5 (this forward-reference is resolved); hosted-serving handshake in plan/TABULAR-API-HANDSHAKE.md. Hosting remains EXECUTION-PENDING REAL COMPUTE — no model served in this environment.
