1.2 Persona split (two tracks, same cadence, one-line swap)

  • Economic buyer (CFO / COO / VP-Ops): copy as written below — opens and WIIFTs framed on cost lines (re-keying hours, exception cost, reconciliation breaks, dispute backlogs).
  • Technical buyer (CIO / VP-Eng / Head-of-Automation): swap the email's first sentence for the persona-swap line given per segment, which reframes the same pain as an integration/pipeline problem (primitives as API calls, splitting/validation as first-class operations, model provenance). Body otherwise identical — one idea per email holds for both.
  • Contact sourcing: role-verified via public footprint only. Never fabricate, guess, or pattern-construct a personal email; no scraped/private data (gtm-high-openrate persona rules). If no public business email exists for the role, that contact runs the LinkedIn-only track (T1→T2→T3).

1.3 Entry criteria (all required)

1. Account is on the TAM-1000 list with segment + vertical tag assigned.

2. At least one role-verified persona contact (economic or technical) via a public/role channel.

3. Account passes suppression screen (§5.4).

4. The segment's E1 artifact exists and is approved (artifact backlog, §4) — no wave starts on a promised-but-unbuilt artifact.

1.4 Exit criteria (any one ends the sequence)

  • Positive reply or call booked → exit to sales immediately (deal motion below).
  • MQL threshold hit (§1.5) → handoff packet to sales; remaining touches cancelled.
  • Unsubscribe / hard bounce / "not relevant" → suppress account-contact permanently; account stays for the other persona unless opt-out is account-level.
  • Sequence complete, no engagement → 90-day re-nurture pool (new angle next cycle; never a resend).

1.5 MQL definition + handoff (account-level, per abm-marketing-play §5)

MQL = any of:

  • (a) one positive reply from either persona (artifact request counts), OR
  • (b) ≥2 gated-artifact requests from the same account, any roles, OR
  • (c) 1 artifact request + 1 LinkedIn accept-with-reply from a target persona.

Explicit non-signals (never MQL alone): opens; a single anonymous form-fill with no role match; LinkedIn accept without reply. No vanity metrics as success criteria.

Handoff packet to sales (SDR/AE owns from here): account + segment/vertical; doc-class hypothesis; full engagement trail (which artifacts pulled, which email/touch got the reply, exact words of the reply); suggested first ask = discovery call, run on the P2P/O2C discovery framework mapped to their highest-volume doc class (gtm-high-openrate R6).

1.6 Deal motion this nurture ladders into (so every touch has a destination)

Discovery call (20 min, agenda + 5–7 doc-class questions) → the specific HyperAPI demo on their doc class (e.g., a representative customs packet, remit file, mill cert batch, or lease set they bring) → pricing conversation framed ONLY on the published usage model: $0.05/page; accuracy/SLA per apis.hyperbots.com; volume/enterprise terms sized on the call — never an invented quote, ACV, or discount → mutual close plan with owner/date/success criterion. The ONE ask of this campaign is the discovery call; E1–E3 buy permission for E4 to make it.
