Dataset summary

SAVIOR-Bench v1 — 509 expert-annotated financial documents for OCR

transcription, layout fidelity (PaIRS), and parse-intent extraction.

Inter-annotator agreement 0.761 (fuzzy string similarity over **187

double-annotated pairs**). Five source corpora, six document classes.

Document classes: invoices, packing slips, forms, multilingual scans,

low-quality PDFs, complex tables.

Tasks & metrics

parse (DC-000) is one task, three facets; only the extraction facet

has data in v1.

  • Extraction (MEASURED): word-level multiset recall / precision;

parse-intent F1 ↑ / WER ↓; OCR normalized edit distance ↓.

  • Layout awareness (DEFINED, NOT YET MEASURED): layout-element accuracy,

reading-order accuracy, PaIRS (PC-only; NPC = format-compat proxy).

  • Bounding box (DEFINED, NOT YET MEASURED): bbox IoU, field-to-region

grounding accuracy.

  • Decoding (frozen, DC-000): temp 0.1, top-p 0.9, max_tokens 2048,

seed 20260516. No number is fabricated for an unmeasured facet.

  • ✗ Degradation / robustness tracks — none in v1 (§8 #5).
  • ✗ Confidence intervals — ci95=null, "reported (no CI)" (§8 #3).
  • ✗ Regenerable / source-traceable corpus — static attested set, mixed

public-corpus licensing (§8 #7).

  • ✗ Numbers reproduced under a fixed harness — attested, not

factory-reproduced (§8 #3; W6 = 0/23 green).

  • PaIRS is PC-only (§8 #2); NPC PaIRS is a format-compatibility proxy.
  • n asymmetry — OCR n=509; parse-intent n unknown (§8 #6).

Marking these ✗ rather than papering over them is the point. Closing R1–R6 is

the substantive path to deeper evidence; until then SAVIOR-Bench claims

format discipline and honest disclosure, not evidence-depth parity.
