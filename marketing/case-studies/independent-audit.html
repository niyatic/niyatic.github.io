1. Snapshot cadence timeline

The file is NOT a constant-cadence series. Monthly through 2024, then weekly from 2025.

2023  May─Jun─Jul─Aug─Sep─Oct─Nov─Dec      (monthly, month-end)
2024  Jan─Feb─Mar─Apr─May─Jun─Jul─Aug─Sep─Oct─Nov─Dec  (monthly)
2025  Jan‧‧‧ → shifts to WEEKLY (Sunday cadence) ‧‧‧‧‧‧‧‧‧‧‧‧‧‧‧‧
2026  ‧W‧W‧W‧W‧W‧W‧W‧W‧W‧W‧W‧W‧W‧W‧W‧W‧W  → 2026-04-26 (latest)
      └────────────── 52 snapshots total ──────────────┘

Implication: all time-series charts must use true calendar dates on the x-axis. A 52-point index plot compresses 18 monthly points and 34 weekly points into equal width and distorts the trend.

Source: df.groupby(df.date.dt.to_period("M")).size() over the converted frame.

7. Bucket-migration mechanism

The file carries Prior Date + Prior Balance per row, enabling defensible week-over-week migration detection without external data:

Snapshot N-1            Snapshot N
┌──────────┐            ┌──────────┐
│ Current  │──stays────▶│ Current  │   (good)
│ 31-60    │──ages─────▶│ 61-90    │   (watch)
│ 61-90    │──ages─────▶│ 91-120   │   (FLAG: new to >90)
│ 91-120   │──pays─────▶│ Current  │   (recovery)
│ Over 360 │──grows────▶│ Over 360 │   (FLAG: write-off candidate accreting)
└──────────┘            └──────────┘

The "new to >90" and "Over-360 growing" transitions are the highest-value signals in the dataset and drive the watchlist panel (dashboard plan Step 8).

The negative-balance vs IsUnappliedCash mismatch (112,363 vs 113,322 — within ~1%) means some negative-balance rows are NOT flagged as unapplied cash → genuine open credits booked in aging rows. Treat balance < 0 as the canonical credit signal; net per customer before presenting.

10. Reproducibility

Every number above is reproducible with this snippet:

import pandas as pd
df = pd.read_excel(
    "aging-source-anonymized.xlsb",
    sheet_name="RawData", engine="pyxlsb", header=3,
).iloc[:, 0:18]
df.columns = ["company","cgid","cgroup","bucket","unappcash","dser","bal","inv",
              "pdser","pbal","pinv","cgclean","pastdue","o60","o90","o120","rank","rankpct"]
df = df[df["company"].notna()]
df["bal"]  = pd.to_numeric(df["bal"], errors="coerce")
df["date"] = pd.Timestamp("1899-12-30") + pd.to_timedelta(
                 pd.to_numeric(df["dser"], errors="coerce"), unit="D")

latest = df["date"].max()
dl  = df[df["date"] == latest]
o90 = dl[dl["o90"] == True]

dl["bal"].sum()                                  # 71,852,567.94
o90["bal"].sum()                                 # 18,246,759.85  (25.4%)
df["date"].nunique()                             # 52
df["cgid"].nunique(), df["cgroup"].nunique()     # 7283, 6722
(df["bal"] < 0).sum(), df["bal"].isna().sum()    # 112363, 6463
dl.groupby("cgclean")["bal"].sum().nlargest(10)  # top-10 by balance
o90.groupby("cgclean")["bal"].sum().nlargest(10) # top-10 by >90
dl.groupby("bucket")["bal"].sum()                # bucket distribution
dl.groupby("company")["bal"].sum()               # entity distribution

Pinned versions: pandas, pyxlsb. Excel serial epoch: 1899-12-30 (Excel 1900 date system with the standard 2-day offset). No seeds required — every operation is deterministic.
