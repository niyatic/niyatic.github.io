HyperAPI vs the document-AI category — a metric-coverage framework

This post is a *framework*, not a scoreboard. It lays out the dimensions on which

document-AI systems should be compared, and is explicit about which cells we can

fill with a sourced number today and which are PENDING MEASUREMENT. We will not

fabricate a single comparative number to make a cell look full.

The honest ground rules

1. Any cell that claims "HyperAPI scores X" must be a sourced SAVIOR-Bench cell

(attested tier, ci95=null) — and labelled as such.

2. Any cell that claims "competitor Y scores Z" must be traceable to that

competitor's own public documentation with a URL — or it is PENDING.

3. There is no "HyperAPI beats the LLM-document-parsing API by N%" cell anywhere in this post.

Producing one requires a head-to-head run on a shared dataset under a pinned

harness. That run has not happened. The honest label is PENDING MEASUREMENT.

The comparison dimensions

A document-AI system should be compared on these axes. For each, we state the

metric and the current data status.

Dimension 1 — OCR transcription accuracy

  • Metric: word-level multiset recall and precision.
  • HyperAPI / Hyperbots data: SAVIOR-Bench v1, n = 509. The fine-tuned

Qwen2.5-VL-7B-F row posts recall 0.9294, precision 0.9157 (attested

2026-05-18, ci95=null, fine-tuned [FT] — not apples-to-apples vs zero-shot).

  • Competitor (LLM-document-parsing vendors) data: PENDING MEASUREMENT. the LLM-document-parsing API's public

materials describe an OCR/parsing pipeline; we have not verified a

word-level recall/precision number on a shared dataset. See DATA-NEEDED.md.

Dimension 2 — Layout fidelity

  • Metric: PaIRS (pairwise spatial-relationship score). Comparable only

between prompt-controlled systems; a proxy otherwise.

  • HyperAPI / Hyperbots data: SAVIOR-Bench fine-tuned rows score PaIRS ~0.707;

strong zero-shot systems lead at ~0.877 (attested, ci95=null). Honest read:

fine-tuning did not win this dimension.

  • Competitor data: PENDING MEASUREMENT.

Dimension 3 — Structured / parse-intent extraction

  • Metric: field-level F1 (↑), word error rate / WER (↓).
  • HyperAPI / Hyperbots data: hyperapi-parse-intent [FT] leads the

SAVIOR-Bench parse-intent panel at F1 0.8328 / WER 0.2589, vs a best documented

zero-shot anchor at F1 0.8052 / WER 0.4232 (attested, n not reported,

ci95=null). The F1 lead is +0.0276 — narrow; the WER reduction is the more

substantive part.

  • Competitor data: PENDING MEASUREMENT.

Dimension 4 — Line-item / table fidelity

  • Metric: row-level F1 (a row is correct only if its cells are correct *and*

bound to the right logical row).

  • HyperAPI / Hyperbots data: PENDING MEASUREMENT as a standalone row-F1

cell — SAVIOR-Bench v1 does not isolate a published row-F1 column.

  • Competitor data: PENDING MEASUREMENT.

Dimension 5 — Document-quality robustness

  • Metric: accuracy stratified by clean-digital / scanned-high / scanned-low /

mobile-capture / multilingual / handwritten-annotated.

  • HyperAPI / Hyperbots data: PENDING MEASUREMENT as a published stratified

grid — this is a convention for future ranking-grade runs.

  • Competitor data: PENDING MEASUREMENT.

Dimension 6 — Cost per correct extraction

  • Metric: observed spend ÷ count of extractions that passed scoring.
  • HyperAPI / Hyperbots data: the comparative study reports an

infrastructure-derived ~$0.19 per 1,000 successful calls for the

fine-tuned inference path (A100 40 GB at ~$2/hr). This is a model-serving cost,

not a Hyperbots product price. See post 09.

  • Competitor data: PENDING MEASUREMENT — competitor pricing is published

per-page/per-token but a cost-*per-correct-extraction* number requires running

their system on a shared dataset.

Dimension 7 — Reproducibility / evidence tier

  • Metric: is every number backed by a rerunnable command on pinned

model/dataset/harness?

  • HyperAPI / Hyperbots data: SAVIOR-Bench v1 numbers are **attested, not

factory-reproduced** (W6 audit: 0/23 parse-intent cells re-derived, for

structural reasons). Honest tier: attested.

  • Competitor data: most vendor accuracy claims ship with no rerun path;

treat as unverified. See post 08.

What a real head-to-head would require

To turn the PENDING cells into sourced cells:

1. A shared, held-out dataset (released after every model's training cutoff).

2. An adapter for each competitor system so the factory — not the vendor —

runs it.

3. Identical decoding per task family across all systems.

4. Per-document score vectors so a bootstrap 95% CI can be computed.

5. A contamination audit per (system, dataset) pair.

6. CI-overlap tie-bands instead of strict ranking.

Until that exists, this post stays a framework. That is the discipline.
