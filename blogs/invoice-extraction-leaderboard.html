Invoice Extraction — Consistency vs the HyperAPI Deployed Extractor STP invoice set

A consistency study — how closely other systems reproduce the HyperAPI deployed extractor's output on a set of straight-through-processed invoices. This is not an accuracy leaderboard and there is no measured-accuracy ranking · generated 2026-05-22

Related: claude-compare leaderboard · Azure DI invoice study · Benchmark Factory overview

Read this first — what this study is, and what it is not. This study measures agreement with HyperAPI's deployed-extractor output on a set of straight-through-processed (STP) invoices — i.e. how closely each other system reproduces HyperAPI's extractions. It is not a measurement of correctness against independent validated truth. The ground-truth file is results.zip, which is HyperAPI's own deployed-extractor output, so HyperAPI is the reference, not a ranked winner — a system agrees with itself by construction. A high agreement number means a system is consistent with HyperAPI's output; it does not mean either system is accurate. For accuracy against independent validated gold, see the Azure DI study's Mini Pilot section and the claude-compare leaderboard.
STP invoice set. Every document in this set is a straight-through-processed (STP) invoice — an invoice the HyperAPI extractor processed end-to-end with no human touch. Because these invoices were handled fully automatically by the deployed extractor, the extractor's output on them (results.zip) is exactly what was used downstream, and that is what every other system is compared against here.

Consistency vs. the deployed HyperAPI extractor agreement, NOT accuracy

What this measures. Every number below is measured against results.zip, which is HyperAPI's own deployed-extractor output on the STP invoice set. So these numbers answer one question: "how closely does another system reproduce what HyperAPI's extractor already produced" — an agreement / consistency metric, not correctness. HyperAPI is the reference by construction: it agrees with itself, so its row is the baseline, not a competitive score. For accuracy against independent validated gold, see the Azure DI study's Mini Pilot section and the claude-compare leaderboard.

Why this consistency study is worth publishing

  • It is a real, reproducible measurement — just of the right thing. "How close is a general-purpose LLM to a deployed production extractor's output on STP invoices" is a legitimate question; it is simply not an accuracy ranking.
  • HyperAPI cannot be ranked here. The ground truth is HyperAPI's own output, so its agreement is 100% by construction. That 100% is the reference baseline — it is not a measured accuracy figure and not a benchmark win.
  • Where a system and HyperAPI disagree, this study cannot tell you who is right. Only an accuracy study against independent validated gold can — see the Azure DI study's Mini Pilot section and the claude-compare leaderboard. A consistency study and an accuracy study answer different questions and must not be merged into one ranking.

Integrity rules applied to this page

  • This is a consistency study, not an accuracy leaderboard. Every number is agreement with the HyperAPI deployed-extractor reference on the STP invoice set — banner-labelled and column-labelled as such.
  • HyperAPI's 100% is not a win. It is shown as "100% (reference)" with an inline explanation: the ground-truth file is HyperAPI's own deployed output, so HyperAPI agrees with itself by construction. No agreement metric is presented as accuracy.
  • For accuracy, look elsewhere. Accuracy against independent validated gold lives in the Azure DI study's Mini Pilot section and the claude-compare leaderboard — not on this page.
  • Real numbers only. Every cell is a measured scorer output — the Claude rows from comparative/results-gt-bench/gt_scores.json, the Azure DI row from comparative/azure-di-consistency/azure_gt_score.json (produced by score_azure_consistency.py, which reuses the same score_gt.py normalizer and canonical field set). Nothing was fabricated for this page. This is the W6 / no-faked-cell discipline.
  • The Azure DI row is the full run. All 200 consistency-study docs are scored: 167 extracted by Azure DI and 33 non-PDF files (Excel/Word mislabeled .pdf) that Azure DI rejects with InvalidContent.
