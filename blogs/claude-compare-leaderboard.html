Read this first — what this page is, and what it is not. This page measures agreement with the Hyperbots deployed extractor's output on the manifest document set — i.e. how closely Claude Opus reproduces what the Hyperbots extractor already produced. It is not a measurement of correctness against independent validated truth. The reference file is results.zip, which is Hyperbots' own deployed-extractor output (HyperAPI's deployed extraction), so Hyperbots is the reference, not a ranked winner — a system agrees with itself by construction. A high agreement number means Claude Opus is consistent with Hyperbots' output; it does not mean either system is accurate. True accuracy requires an independent ground-truth file, which is still pending. For the companion consistency study on the STP invoice set, see the Invoice Extraction Leaderboard.
Why a consistency study. "How close is a general-purpose LLM to a deployed production extractor's output" is a legitimate, reproducible question. It is simply not an accuracy ranking: where Claude Opus and Hyperbots disagree, this study cannot tell you which one is right — only an accuracy study against independent validated gold could. That independent ground-truth file does not exist for this set yet.

Honest framing note (2026-05-23 cross-model head-to-head). This is a consistency study: agreement with HyperAPI prod reference (results.zip), NOT independent accuracy. The surprising-but-real finding: opus is the lowest here because opus extracts MORE fields than the lean prod reference records — those extras land as false positives. Gemini-pro / Gemini-flash / haiku produce leaner output that aligns more closely with the prod reference and score higher on agreement. The dataset_95 mini-pilot (real granular gold ground truth) shows the opposite ordering — opus 76.10%, HyperAPI 35.73%. Two different things being measured. Don't read this table as "haiku > opus on quality."

Where Claude Opus and Hyperbots agree — and where they diverge

  • High agreement on summary financial totals. invoice_total F1 0.965, tax_total 0.975, total_amount 0.940, currency 0.897 — Claude Opus closely reproduces the Hyperbots extractor's header numbers.
  • Line-item numerics agree well. quantity 0.929, unit_price 0.889, line amount 0.827, line tax 0.842 — and line-item-count exactly matched on 89.3% of documents.
  • Line-item description is the weakest field (F1 0.469). Claude Opus often emits a normalized or paraphrased description where the Hyperbots extractor emits verbatim text (and vice versa). This is a real measured divergence — under a token-set Jaccard ≥ 0.6 match it is counted honestly as both an FP and an FN. It is a disagreement, not evidence either side is wrong.
  • amount_paid agreement is low (F1 0.239, recall 0.138). The Hyperbots reference carries an amount_paid_since_last_invoice value on most docs; Claude Opus emits it on only ~14%. Counted as honest recall misses (FN) — never fabricated.
  • This study cannot adjudicate the disagreements. Where the two diverge, neither is declared correct — there is no independent ground truth here.

Reproduce

Stdlib-only, deterministic. Document-level bootstrap: 1,000 resamples, seed 42. Numeric tolerance: max(1% relative, 0.01 absolute). Description match: token-set Jaccard ≥ 0.6.

python3 comparative/results-gt-bench/score_gt_manifest.py # scores all 4 Claude-opus batches (n=400) vs results.zip and writes # /tmp/gt_scores_manifest.json (reuses the score_gt.py normalizer)
