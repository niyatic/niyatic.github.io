6 · Reproducibility & audit

W6 attestation probe — 0 / 23 cells factory-re-derived.
The W6 workstream (run 2026-05-20, harness @ 3382abb) attempted to re-derive every cell of the 23-row DC-000 parse-intent-inhouse-v1 table under the pinned offline harness. Outcome from results/_w6_factory_rerun.json:
  • cells attempted: 23 · green repro checks: 0 · CIs added: 0 · attested values retracted: 0
  • Every rerun returned code 1 — red reason no-adapter-for-vendor-model-in-SEED-harness for all 23.
Why (structural): (1) the harness adapter REGISTRY registers only SEED adapters (rule-baseline-v1, keyword-extractor-v1, hyperbots-ip-seed-v1) — none of the 23 vendor models has an adapter; (2) run.py LOADERS does not register parse-intent-inhouse-v1 — the in-house corpus is not in-repo; (3) the attested benchmark.xlsx carries only aggregate F1/WER per model, no per-document scores — nothing to bootstrap a CI from. The attested numbers are preserved unchanged (not contradicted, only un-re-derivable here), reproduced_by_factory stays false, ci95 stays null. No interval was fabricated.

Repro check (synthetic-fixture cells)

From results/_repro_check.json: the deterministic harness re-ran a sample of fixture cells and confirmed all_match: true (sampled 2 of the 12 fixture cells; metric and 95% CI matched exactly, same git_commit 97ee05a). Fixture cells are reproducible byte-for-byte from pinned seed 20260516.

Contamination audit

From contamination/audit-records.json (3 records, one per fixture-cell model): finbench-synth-fixture-v1 is structurally-none leakage risk — generated deterministically at run time from pinned seed 20260516, sha256 frozen, never published before the run; no model could have trained on it.

The attested / external datasets (savior-bench-v1, parse-intent-inhouse-v1, ocr-edit-distance-v1, financepss-205-v1) have no contamination audit record — contamination status for those is not assessed in this run.
