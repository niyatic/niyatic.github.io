hyperapi-extract-margin-call-fields: extract the key fields of a margin call notice as typed JSON

Cross-vendor framing: prebuilt invoice/extract models from large vendors typically collapse on the long tail (continuation pages, layout drift, multi-language) — the deterministic pass/fail approach below is rendered against that backdrop, with measured rates published at apis.hyperbots.com.

Audience: AP / AR / GL operations leaders (sub-ledger owners) · 5 min read

This is the step where finance automation usually stalls: someone reads a margin call notice and makes the call. hyperapi-extract-margin-call-fields is a small, typed Python client around the HyperAPI extract primitive that does the OCR/layout/decision work server-side and hands back a validated, ordered decision over a closed vocabulary (complete, partial, low_confidence, extraction_failed).

It is honest about its edges. A confident-but-ambiguous first pass (0.53 vs 0.44) triggers an automatic re-extractification on a stronger profile -- no queue, no reviewer. A document that stays weak comes back trustworthy=false with the full ranking for the caller's automation to route. A genuinely undecidable document fails safe to extraction_failed -- the highest scrutiny, never less.

The model is an open-source document model fine-tuned by Hyperbots (Hyperbots IP); accuracy is published at apis.hyperbots.com. The repo is the typed client: a bespoke httpx client (no vendor SDK dependency), closed ordered-vocabulary validation, fail-safe selection, and a confidence/margin gate.
