Classify which insurance document class a file belongs to: typed result in one call

Insurance intake is a mixed stream of policy, certificate, and claim documents. This package classifies each into a closed insurance document class as typed JSON.

The hard part

It is the borderline document -- a poor scan or an ambiguous case. This package gates on confidence and the top-vs-runner-up margin, so the borderline case is caught and automatically re-run on a stronger profile.

Fails safe, never a queue

When the model genuinely cannot decide, the result fails safe to the highest-scrutiny outcome. Documents that stay weak come back trustworthy=false for your automation to route by rule. It is never queued to a person.

Five lines

``python from hyperapi_classify_insurance_document_type import analyze_file r = analyze_file("data/sample/invoice.pdf") print(r.decision.label, r.decision.confidence, r.trustworthy) ``

Accuracy is published at apis.hyperbots.com. The model is an open-source document model fine-tuned by Hyperbots (Hyperbots IP); this repo is the typed client around it.
