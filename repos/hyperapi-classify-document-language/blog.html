Pick the locale before you extract, not after

A finance pipeline that assumes every document is en-US will silently parse a French 1.234,56 as one thousand two hundred — and a UK 03/04/2025 as March instead of April. By the time a downstream validator notices, the number is already wrong in three systems.

hyperapi-classify-document-language runs one HyperAPI classify call and returns a typed Language, an optional region, a calibrated confidence (surfaced verbatim from the API), and an ExtractionLocale hint that tells the next stage which date order and decimal convention to expect. Below a tunable confidence threshold — or on a bilingual document where the model is nearly split — the document is *not* guessed; it is re-submitted (confidence-gated automatic re-extraction). No side path.

The language/locale model is an open-source model fine-tuned by Hyperbots on multilingual finance corpora (Hyperbots IP). Accuracy is published per apis.hyperbots.com.
