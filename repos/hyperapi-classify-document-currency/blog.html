Know an invoice's currency before it hits the ledger

Every intake pipeline hits the same wall before it can do anything useful: A USD invoice posted to a EUR entity is a reconciliation fire drill weeks later. The currency is on the page; reading it reliably across templates and "$"-shared symbols is the work.

hyperapi-classify-document-currency turns that into one function call that returns a typed result — which ISO-4217 currency a finance document is denominated in — with a confidence score on every class.

The hard part isn't the obvious case

It's the near-tie. Two classes a few points apart means the document reads as both, and a single top score hides that. This package gates on the margin to the runner-up as well as the top score, and runs a deterministic cross-signal self-check so a currency that contradicts its own symbol/locale marker is caught before it routes.

No queue, just a flag

A borderline first pass is automatically re-classified on a higher-resolution profile; the stronger pass is kept. Documents that stay weak come back trustworthy=false with the scores surfaced, for your automation to route by rule. Routing stays in your automation, end to end.

Five lines

``python from hyperapi_classify_document_currency import classify_file r = classify_file("document.pdf") print(r.classification.scores.top(), r.trustworthy) ``

Accuracy is published at apis.hyperbots.com. The model is an open-source document model fine-tuned by Hyperbots (Hyperbots IP); this repo is the typed client around it.
