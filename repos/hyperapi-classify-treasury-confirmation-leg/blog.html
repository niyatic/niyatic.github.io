A treasury confirmation into a closed settlement-leg vocabulary -- with no person in the loop

hyperapi-classify-treasury-confirmation-leg is a small, typed Python client around the HyperAPI classify primitive. It does the OCR, layout, and decision work server-side and hands back a validated, ordered result with a deterministic fail-safe.

The interesting part is what happens when a document is hard: there is no review queue. A weak first pass is automatically re-processed by the service on a higher-resolution profile; the stronger pass wins; a genuinely undecidable document fails safe to the highest-scrutiny outcome. The result is always returned, flagged for the caller's automation to route.

See the repo README.md and examples/hard_document_walkthrough.md for a fully mocked, reproducible end-to-end run.
