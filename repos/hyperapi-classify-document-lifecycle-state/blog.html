Stop paying drafts and voids: lifecycle state in one call

The same invoice means four different things depending on where it is in its life: a draft proforma creates no obligation, an issued invoice is payable, a paid copy is closed, a voided one must never be paid. Miss the state and you pay a draft, double-pay a settled invoice, or honour a cancelled one. hyperapi-classify-document-lifecycle-state turns that fork into one function call that returns a typed label with a confidence score on every class.

The hard part isn't OCR

The template barely changes between states -- the signal is a watermark, a paid stamp, a zero balance, a VOID overlay. The library returns the on-document state marker the model read and runs a deterministic marker_consistent() check -- a high-confidence state flip is caught even when the score looked clean.

No queue, just a flag

A borderline first pass -- low top score *or* a thin margin to the runner-up *or* a marker conflict -- is automatically re-classified on a higher-resolution profile; the stronger pass is kept. Documents that stay weak come back trustworthy=false for your automation to route by rule. Routing stays in your automation, end to end.

Five lines

``python from hyperapi_classify_document_lifecycle_state import classify_file r = classify_file("document.pdf", org_aliases=["Contoso Manufacturing Inc"]) print(r.classification.label, r.trustworthy) ``

Accuracy is published at apis.hyperbots.com. The classification model is an open-source document model fine-tuned by Hyperbots (Hyperbots IP); this repo is the typed client around it.
