hyperapi-classify-margin-call-urgency: classify the urgency of a margin call as typed JSON

This is the step where finance automation usually stalls: someone reads a margin call notice and makes the call. hyperapi-classify-margin-call-urgency is a small, typed Python client around the HyperAPI classify primitive that does the OCR/layout/decision work server-side and hands back a validated, ordered decision over a closed vocabulary (routine_call, same_day_call, immediate_call, undetermined_urgency).

It is honest about its edges. A confident-but-ambiguous first pass (0.53 vs 0.44) triggers an automatic re-classifyification on a stronger profile -- no queue, no reviewer. A document that stays weak comes back trustworthy=false with the full ranking for the caller's automation to route. A genuinely undecidable document fails safe to undetermined_urgency -- the highest scrutiny, never less.

The model is an open-source document model fine-tuned by Hyperbots (Hyperbots IP); accuracy is published at apis.hyperbots.com. The repo is the typed client: a bespoke httpx client (no vendor SDK dependency), closed ordered-vocabulary validation, fail-safe selection, and a confidence/margin gate.
