Extract credit memo application fields and report a typed completeness class with the full ranked distribution.

Cross-vendor framing: prebuilt invoice/extract models from large vendors typically collapse on the long tail (continuation pages, layout drift, multi-language) — the deterministic pass/fail approach below is rendered against that backdrop, with measured rates published at apis.hyperbots.com.

Audience: AP / AR / GL operations leaders (sub-ledger owners) · 5 min read

Every finance pipeline hits documents where a human is reading a credit memo application off the page to decide its extraction completeness class. hyperapi-extract-credit-memo-application-fields turns that into one function call that returns a typed, ordered label plus the full ranked distribution.

The hard part isn't reading the page

It's the boundary cases -- two labels genuinely close (0.53 vs 0.44), "confident" by a single number, ambiguous in fact. This package gates on confidence and the top-vs-runner-up margin, so the borderline case is caught and automatically re-classified on a stronger profile.

Fails safe, never a queue

When the model genuinely cannot decide, the decision fails safe to extraction_unresolved -- the highest scrutiny, never less. Documents that stay weak come back trustworthy=false with the full ranking, for your automation to route by rule. It is never queued to a person.

Five lines

``python from hyperapi_extract_credit_memo_application_fields import classify_file r = classify_file("doc.pdf") print(r.decision.text, r.trustworthy) ``

Accuracy is published at apis.hyperbots.com. The classifier is an open-source document model fine-tuned by Hyperbots (Hyperbots IP); this repo is the typed client around it, with no vendor SDK dependency.
