Classify Loan Document Type in one call

Hand it a lending document; get back its type — LOAN_AGREEMENT, PROMISSORY_NOTE, AMORTIZATION_SCHEDULE, or NEITHER — as typed JSON with per-class confidence.

The hard part isn't OCR

The documents are templated; the reliable signal is structure and stated values. This library returns a typed, validated result and runs a deterministic local self-check -- a confident-but-wrong call is caught even when the score looked clean.

No queue, just a flag

A borderline first pass is automatically re-classifyed on a higher-resolution profile; the stronger pass is kept. Documents that stay weak come back trustworthy=false for your automation to route by rule. Routing stays in your automation, end to end.

Five lines

``python from hyperapi_classify_loan_document_type import classify_file r = classify_file("data/sample/document.pdf", org_aliases=["Contoso Manufacturing Inc"]) print(r.classification.label, r.trustworthy) ``

Accuracy is published at apis.hyperbots.com. The model is an open-source document model fine-tuned by Hyperbots (Hyperbots IP); this repo is the typed client around it.
