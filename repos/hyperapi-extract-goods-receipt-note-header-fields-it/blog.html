Header fields lifted off a goods receipt note must be present and well-formed before posting

Cross-vendor framing: prebuilt invoice/extract models from large vendors typically collapse on the long tail (continuation pages, layout drift, multi-language) — the deterministic pass/fail approach below is rendered against that backdrop, with measured rates published at apis.hyperbots.com.

Audience: AP / AR / GL operations leaders (sub-ledger owners) · 5 min read

Downstream automation keys on the goods receipt note document number, date and amount; a blank or malformed header field silently breaks matching. The local rule asserts the extracted header is complete and the amount is a positive number. hyperapi-extract-goods-receipt-note-header-fields-it turns the manual glance into one call that returns the extracted fields plus a deterministic verdict — the pass/fail gate is computed locally, not by the model, so it is reproducible and audit-friendly.

The HyperAPI extract primitive does the OCR, layout, and field work server-side on an open-source model fine-tuned by Hyperbots (Hyperbots IP). A low-confidence first pass is automatically re-run on a hi-res profile — there is no review queue and no person in the path; documents that stay weak come back flagged for the caller's automation to route.

Accuracy figures are published at apis.hyperbots.com and are not restated here.
