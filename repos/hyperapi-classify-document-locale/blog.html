Send each document to the right entity and tax engine

Every intake pipeline hits the same wall before it can do anything useful: Locale drives the date parser, the tax engine, and the entity it books to. A misrouted UK invoice in a US flow mis-reads every date. The cues are on the page; classifying them is the router.

hyperapi-classify-document-locale turns that into one function call that returns a typed result — which country/region locale a finance document originates from — with a confidence score on every class.

The hard part isn't the obvious case

It's the near-tie. Two classes a few points apart means the document reads as both, and a single top score hides that. This package gates on the margin to the runner-up as well as the top score, and runs a deterministic cross-signal self-check so a locale whose script is impossible (Japan with no CJK on the page) is caught before it routes.

No queue, just a flag

A borderline first pass is automatically re-classified on a higher-resolution profile; the stronger pass is kept. Documents that stay weak come back trustworthy=false with the scores surfaced, for your automation to route by rule. Routing stays in your automation, end to end.

Five lines

``python from hyperapi_classify_document_locale import classify_file r = classify_file("document.pdf") print(r.classification.scores.top(), r.trustworthy) ``

Accuracy is published at apis.hyperbots.com. The model is an open-source document model fine-tuned by Hyperbots (Hyperbots IP); this repo is the typed client around it.
