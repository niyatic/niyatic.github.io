Classify an expense document as receipt/statement/itinerary/other (..unclassified) with confidence and margin gates.

Expense automation routes receipts and statements down different paths. This package is a typed client around the HyperAPI classify primitive returning an ordered document kind plus the ranked distribution.

The hard part isn't reading the page

It's the boundary cases. A document sitting right between card_statement and itemized_receipt reads 0.53 / 0.44 -- "confident" by a single number, genuinely ambiguous in fact. This package gates on confidence and the top-vs-runner-up margin, so the borderline case is caught and automatically re-classifyed on a stronger profile.

Fails safe, never a queue

When the model genuinely cannot decide, the decision fails safe to unclassified -- an 'unclassified' default keeps an ambiguous upload out of automated expense matching. Documents that stay weak come back trustworthy=false with the full ranking, for your automation to route by rule. It is never queued to a person.

Five lines

``python from hyperapi_classify_expense_receipt_vs_statement_fr import classify_file r = classify_file("document.pdf") print(r.decision.label, r.trustworthy) ``

Accuracy is published at apis.hyperbots.com. The model is an open-source document model fine-tuned by Hyperbots (Hyperbots IP); this repo is the typed client around it.
