Augment (O2C) — deep-study plan

1. Executive summary (TEMPLATE — numbers placeholder)

Augment for O2C enriches extracted payment / statement / remittance fields by lookups: customer master (canonicalise customer name → ID + credit terms + open AR balance), open-invoice subset retrieval (which invoices are eligible to receive this payment), deduction-code master (canonical code + authority-limit), bank/lockbox master (which payment channel), FX rates, holiday calendars, and customer-credit-tier classification (for downstream credit-limit checks). Output = enriched field set + per-enrichment confidence + provenance. Numbers [PENDING-MEASUREMENT].
