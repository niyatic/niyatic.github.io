Split (P2P) — deep-study plan

1. Executive summary (TEMPLATE — numbers placeholder)

Given a multi-document P2P bundle (a single PDF containing e.g. PO + multiple GRNs + supplier_invoice + BoL + packing_slip), how correctly does an IDP system identify the page boundaries between logical sub-documents? Distinct from classify (which assigns a label once the boundary is set) and from parse (which extracts within a sub-document). Output = an ordered list of (start_page, end_page) spans + optionally a per-span label. Numbers [PENDING-MEASUREMENT].

2. Scope & coverage

  • Workload. Multi-doc bundles drawn from P2P workflows: AP intake

bundles (one supplier email = one PDF with cover-page + invoice + GRN + BoL + packing slip), brokerage bundles (BoL + customs forms + freight invoice).

  • Platforms. Hyperbots split + ≥1 competitor.
  • Coverage dims. 9 D-dimensions kept (D4, D5, D6, D9 dropped —

orthogonal to boundary detection; D11 reinterpreted as page-attribution accuracy not field IoU).

  • Test surface.
  • n = [PLACEHOLDER] bundles per dimension.
  • Stratified by bundle page count (2–5 / 6–15 / 16–50 / 50+), by

sub-document count (2 / 3–5 / 5+), by inter-document layout transition style (cover-page / no-cover-page / continuation-of-table- across-boundary), and by file format.

3. Methodology

3.1 Data-status representation

Eight-row legend per framework §0.4. Per-bundle: Correct = predicted spans set-equal to GT spans; Incorrect = predicted span overlaps GT but not exact; Missed = GT span absent from prediction; Spurious = predicted span absent from GT.

3.2 Sample design

n = [PLACEHOLDER] bundles per dim, stratified per §5. κ on bundle-level boundary annotation (two annotators independently mark page boundaries; disagreements adjudicated).

3.3 Metrics

  • Boundary-F1. Treat each page-N-to-page-N+1 transition as a binary

"is this a boundary?" prediction; report per-bundle precision/recall and aggregate F1.

  • Span-F1. Predicted (start,end) span vs GT span — exact-match

precision/recall + IoU-based partial credit.

  • Page-attribution accuracy. Per-page: did this page land in the

right sub-document?

  • Full-bundle exact-match accuracy (strict): fraction of bundles

where the entire span list matches GT.

  • Point + 95% bootstrap CI + n.

3.4 Scoring rubric

  • Boundary at position k is Correct iff GT also has a boundary at k.
  • Page-attribution: page p is Correct iff predicted sub-doc for p ==

GT sub-doc.

  • Off-by-one boundaries are NOT half-credit — they are Incorrect at the

boundary level; the page-attribution metric is what captures "almost-right."

3.7 Limitations

  • D4 / D5 / D6 / D9 do not apply at the boundary level.
  • Continuation-of-table-across-boundary strata are genuinely ambiguous

(is a continued table one logical doc or two?). The pinned policy is documented in PINS.txt and may produce wider CIs in that stratum.

3.8 Where this primitive sits in the IDP pipeline

The split stage of parse → split → classify → extract → augment → validate → match → GL → reason. Upstream: parse (page-level OCR). Downstream: classify and extract operate on the spans this stage produces.
