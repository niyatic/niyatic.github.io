Classify (O2C) — deep-study plan

1. Executive summary (TEMPLATE — numbers placeholder)

Assign one of eight O2C labels — sales_order / sales_invoice / asn / customer_credit_memo / dunning_notice / statement_of_account / remittance_advice / dispute_correspondence — plus an other reject class. O2C classification has its own confusability pairs: remittance_advice ↔ statement_of_account (both table-heavy), dunning_notice ↔ dispute_correspondence (both prose). Numbers [PENDING-MEASUREMENT].

2. Scope & coverage

  • Workload. Eight O2C labels + reject.
  • Platforms. Hyperbots classify + ≥1 competitor.
  • Coverage dims. 11 of 14 (D4, D11 dropped).
  • Test surface.
  • n = [PLACEHOLDER] per dim.
  • Stratified by label-frequency-band and by table-heaviness (remittance

and statement carry table-presence-as-feature load — D8*).

  • κ per class.

3. Methodology

3.1 Data-status representation

Eight-row legend per framework §0.4. Class-specific Correct / Incorrect / Missed / Spurious as in classify-P2P §3.1.

3.2 Sample design

n = [PLACEHOLDER] per dim; label-frequency-band stratification with explicit upsampling of dispute_correspondence (free-form prose tail).

3.3 Metrics

  • Per-class P/R/F1; macro-F1.
  • Confusion matrix with off-diagonal entropy.
  • AUROC per class.
  • Precision-at-coverage curve.
  • 95% bootstrap CI + n.

3.4 Scoring rubric

Top-1 exact-label match; reject as first-class label; ε-tie abstention. Pinned ε in PINS.txt.

3.5 Evaluation controls

Frozen sha256. Rerunnable command:


run --study classify-o2c-v0 \
    --dimension D<id> \
    --system hyperbots-idp@<model-revision> \
    --system <competitor>@<api-version> \
    --dataset o2c-classify-bench-v1@<snapshot-sha256> \
    --split test --seed <pinned> \
    --metric macro_f1 --bootstrap 1000

3.6 Independence and disclosure

Hyperbots-published; affiliation declared.

3.7 Limitations

  • dispute_correspondence labels carry residual GT subjectivity (some

emails are ambiguous between "dispute" and "general inquiry"); per-class κ will be lower than for structured O2C docs.

  • Statement vs remittance confusion is the dominant fixable failure mode;

off-diagonal entropy is the right metric for it.

3.8 Where this primitive sits in the IDP pipeline

classify stage; upstream parse/split; downstream extract.
