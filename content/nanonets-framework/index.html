What is reusable (the inheritance contract)

Every cell-plan inherits, unchanged, from the framework:

  • The nine-section spine.
  • The eight-row data-status legend (Correct / Incorrect / Missed /

Spurious / Structure / Assessment in progress / Vendor-reported / Not yet measured).

  • The Δ-reading rule (non-overlapping 95% CIs only).
  • The independence + affiliation disclosure language.
  • The IDP-pipeline-position figure (`parse → split → classify →

extract → augment → validate → match → GL → reason`).

  • The pinned, rerunnable command form.
  • The honesty discipline (no faked cells; [PLACEHOLDER] /

[PENDING-MEASUREMENT] until measured).

What changes per cell: the in-scope D-dimensions, the per-primitive metric specialisations, the doc × failure-mode matrix rows, the limitations, and the falsifiable hypotheses.

How to use this library

  • Per blog post: pick one cell-plan; the cell-plan IS the blog

plan. Editing converts placeholders to measured numbers and adds the per-edition snapshot sha256.

  • Per RFP / customer ask: the framework + relevant cell-plan(s)

answer "how would you measure that?" without invented numbers.

  • Per measurement run: the cell-plan §3.5 carries the one literal

rerunnable command that produces the cell's numbers.
